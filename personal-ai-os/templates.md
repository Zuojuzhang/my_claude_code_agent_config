# 派工模板

版本：v1.0（2026-07-08）

## 弱模型派工设计规则（写任何派工prompt前先看）

1. 一次派一个任务，不打包多任务
2. 全部用绝对路径，禁止「项目里的那个文件」式指代
3. 产出物路径预先指定，不让执行者自己决定放哪
4. 步骤编号，每步有产物，中断可从断点续跑
5. 完成标准优先可机器验证（文件存在、条数相等、字段齐全），主观标准给判例
6. 执行者拿不准的列「待确认项」集中汇报，禁止自作主张选一个
7. 凭证只给存放位置，明文永不进prompt
8. 执行者第一步永远是Read列出的文件，禁止凭训练记忆执行流程

## 模板骨架（所有派工prompt的固定结构）

```
你是<角色>，任务：<一句话>。

## 先读（按序Read，读完再动手）
- <流程文档绝对路径>
- <输入文件绝对路径>

## 已确认的决定（直接采信，禁止重新探索或质疑）
- <人已拍板的结论>

## 执行步骤
1. <步骤> → 产物：<绝对路径>
2. ...

## 禁止
- <红线>

## 完成标准（可验证）
- <标准>

## 汇报格式
产物路径清单 / 完成标准逐条勾选结果 / 待确认项
```

## T1 课件生产（大纲已确认后）

```
你是课件制作者，任务：把<项目名>做成可翻页的网页课件lesson.html。

## 先读（按序Read，读完再动手）
- /Users/zrf/.claude/skills/live-lesson-deck/SKILL.md（流程权威，逐节照做）
- <大纲文件绝对路径>（人已确认的主线）
- <项目目录绝对路径>（课件内容来源）

## 已确认的决定
- 大纲即取舍结论：不在大纲里的主题不做，大纲里每页的主题和顺序不改
- 目标学员：<一句话画像>

## 执行步骤
1. 按大纲逐页制作 → 产物：<项目目录>/lesson.html
2. 浏览器自检：翻页正常、console无报错
3. 按 ~/.claude/personal-ai-os/checklist.md 的B1组七条逐条自检

## 禁止
- 增删页面主题或改页面顺序（发现大纲有问题列进待确认项，不自己改）
- 按项目的制作时间线铺页

## 完成标准
- lesson.html存在且左右键可翻页
- B1七条全过，或列出未过条目和原因

## 汇报格式
产物路径 / B1逐条勾选结果 / 待确认项
```

## T4 微信读书笔记落盘

```
你是笔记归档员，任务：把下面粘贴的微信读书划线整理进知识库。

## 目标目录
/Users/zrf/workspace/personal-note/obsidian-vault/Notes/<YYYY-MM>/

## 规则
- 每本书一个md文件：<书名>-读书笔记.md。文件已存在则在末尾追加新划线，不覆盖已有内容
- frontmatter三字段：created（ISO日期）、tags（含「读书笔记」）、book（书名）
- 每条划线一个bullet；我写的想法用引用块（>）紧跟在对应划线后
- 双链用短路径[[概念名]]，只链vault里已存在的同名笔记，不确定就不加

## 禁止
- 改写划线原文
- 删除文件里已有的任何内容

## 完成标准
- frontmatter三字段齐全
- 落盘划线条数 = 输入条数（汇报里给两个数字对照）

## 汇报格式
产物路径 / 条数对照（输入N条，落盘N条）/ 加了哪些双链

## 输入
<粘贴划线文本>
```

## T5 每周巡库

```
你是知识库管理员，任务：对知识库做每周巡库。

## 目标目录
/Users/zrf/workspace/personal-note/obsidian-vault/

## 执行步骤
1. 找出Notes/下最近7天新增或修改的md文件（按文件mtime）
2. 逐篇检查：frontmatter缺字段的补上；正文里出现的、vault中已有同名笔记的概念加[[双链]]（短路径）
3. 更新Concepts/：新笔记按主题挂到已有Concept索引页；同一主题攒到3篇以上且没有索引页的，新建一页列出这些笔记的链接
4. 巡库报告追加到 obsidian-vault/Concepts/_巡库日志.md（新增几篇、补了几个链、Concepts改动清单）

## 禁止
- 删除或合并任何笔记
- 改写笔记正文（只允许加链接、补frontmatter缺字段）

## 完成标准
- 本周每篇新笔记都出现在至少一个Concept索引页里
- _巡库日志.md有本次记录

## 汇报格式
巡库报告原文
```
