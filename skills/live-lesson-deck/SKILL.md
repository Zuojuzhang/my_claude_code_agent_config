# 正式直播教材化

把一个做完的项目变成正式直播课的课件。输入是项目的真实历史（工作日志、踩坑记录、代码现状）加排课信息，输出是单文件lesson.html落在项目根目录（同项目多课时用lesson-2.html递增），讲者全程对着它讲、按N看提词、带学员做。

## 全流程七步

### 第1步 排课六问（动手前必问）

一次性问齐，每个答案都直接改变deck的结构：

| 问题 | 决定什么 |
|---|---|
| 学员画像（零代码/入门/技术） | 比喻密度、代码出现量、原理讲多深 |
| 课程定位（系列第几课、要回扣哪一课、下一课是什么） | 封面眉题编号、开场回扣、收尾预告 |
| 课堂形态（纯讲/讲+带做/工作坊） | 带做环节数量与占比 |
| 结课目标（听懂/跑通最小链路/完整复刻） | 带做深度、验收清单、翻车预案规格 |
| 配图方式（重绘动效/混合/真实截图） | 视觉路线与文件体积 |
| 排课时长与场次 | 总分钟预算、页数规模、休息插点 |

上下文里已有答案的不重复问；拿不准宁可问，错误的默认假设会让整份deck返工。

被作为subagent一口气外派时（调用方声明了一口气模式，或没有与用户的对话通道）：六问答案由调用方在工作包里给齐，缺的用保守默认并记入待确认项；第4步的确认改为把环节表直接写进交付物，拿不准的集中列「待确认项」，不停下等确认。

### 第2步 挖素材

Read项目的WORKLOG.md和CLAUDE.md，从真实历史里挖教学素材，优先级从高到低：

1. 踩坑与破案过程。完整结构：症状、第一反应（猜错的那个）、关键动作、证据、修复。这是全课最有灵魂的素材，照真实经过讲，不美化
2. 失败的前传。曾经放弃的方向和转向原因，是天然的开场故事和方法论引子
3. 真实数字。对比实验的条数、耗时、比例，数字比形容词有说服力
4. 成品本身。看板、脚本、exe，决定成品先行环节怎么演示

只用项目里真实发生的事。素材单薄的段落如实告诉用户，问有没有补充，不编。

### 第3步 设计课程结构

叙事骨架按「成品先行 → 起点故事 → 原理比喻 → 现场演示 → 学员带做 → 踩坑实录 → 用数据说话 → 收束」走。骨架模板、页型构成、分钟分配参考Read references/page-patterns.md。

结构层规则：
- 方法论显式立「方法卡」，全课3到6张，每张在剧情自然出现处立起，收尾合成一页速查卡。方法卡名字要短，后文能反复回扣
- 章节分钟预算之和等于排课净时长；完整复刻型的带做时间约占一半；中场休息插在第一个带做之后
- 每个环节只压一个教学点，每页只有一个信息焦点
- 互动三件套各就各位：大字提问页放在原理引入处（收学员答案），站队页放在悬念揭晓前（举手表态），庆祝检查点页放在带做完成后（举手统计加拍照）
- 骨架表里标的方法卡插点是最低配置，剧情需要可以加密插卡，总数守住3到6张
- 结课目标与时长冲突时（原项目载体太重，规定时长内该画像学员复刻不可达）：保核心链路，换更轻的载体（如原生App换网页版），降级决策在deck页面里向学员说明，并列入待确认项交用户裁决，不悄悄降

### 第4步 跟用户确认结构

把环节表（环节名/页码范围/分钟/教学点一句话）发给用户过目，标题、关键比喻、带做范围在这一步定下来再动手。deck体量大，结构返工的代价远高于多一轮确认。

### 第5步 建deck

视觉与动效：
- 先加载frontend-design skill，按它的 Design Thinking 给出 Purpose/Tone/Differentiation 三项承诺再动手
- 取值优先级：工作区既有lesson.html范例的排版骨架与动效性格 > 用户点名的配色主题 > frontend-design 的 Aesthetics Guidelines。先Glob工作区（含_archive）找既有范例，有就沿用骨架只换光源与配色，没有再按references/page-patterns.md从零建
- 动效是教学内容的可视化翻译：这页在讲什么关系，动效就演什么关系。对比关系演成左右对峙，流程关系演成路线生长，揭晓关系演成数字滚动，结构关系演成逐段点亮。不从装饰角度挑动效，内容换了动效应当随之失效
- 全deck一种动效性格（一条缓动曲线、一种入场逻辑）；自动播放的演示页进入即播、离开即停；全量reduced-motion降级
- 成品演示页用页内重绘的live demo（假数据自动循环），讲师注记写明现场优先切真程序，deck版是备份
- 浅色主题注意品牌色对比度：图形层（边框/光斑/柱条）用原品牌色，文字层用同色相加深的版本，保证正文4.5比1、大字3比1

交互标准（行为级）：
- 单文件HTML，零运行时依赖，浏览器直接打开；web字体允许外链，但必须带系统字体fallback，断网也能讲
- ←→翻页、空格逐步揭示要点、N呼出讲师注记抽屉、Esc收起、Home/End跳首尾
- 触摸滑动翻页；地址栏井号页码直跳；底部进度条与页码；首次打开有操作提示，操作后自动消失
- 讲师注记抽屉同时显示当前页注记与所在环节的时间预算

讲师注记（每页一条，写给讲者的提词，不是给学员看的）：
- 这页怎么开口、节奏快慢
- 提问怎么发、收几个答案、怎么接住
- 哪句要重读、哪里停顿
- 时间提示（预算几分钟、何时统计进度）
- 现场操作（切真程序、宣布休息、拍照发群）
- 翻车预案（网络不行怎么办、学员环境异常怎么办、现场接口抽风的Plan B）

文案：
- 页面全部文字遵循no_ai_style
- 标题要有钩子，口语化；每页的关键句单独成行，注记里标注重读
- 零代码学员的课，技术名词每课只保留一两个让学员记名字，其余全部用日常事物比喻（快递、电话、查户口这个量级）
- deck里展示的可粘贴提示词必须真实可用，并提醒用户把文本版发到群文件，页面只是展示。会话内没条件实跑验证的提示词，列入课前准备清单并注明：开课前用干净环境各跑一遍

### 第6步 自检

- 自检工具首选playwright脚本（截图+console收集+行为断言）：总页数与注记条数对齐、键盘翻页与抽屉可用、关键页逐张截图目检、console零代码报错
- 内置预览面板的渲染时钟可能冻结（CSS动画停在第一帧、截图超时），那是面板的问题不是deck的bug，以playwright结果为准
- 对照ui_engineering_baseline工程底线：文字对比度、中文断行、装饰图形不压文字、reduced-motion。这份清单是工程门槛的唯一权威，跑一轮即可
- 临时验证脚本与截图验完即删

### 第7步 交付

- 报lesson.html完整路径与操作按键（命名规则：首课lesson.html，同项目后续课时lesson-2.html递增）
- 给课程结构表：环节/页码/分钟/教学点
- 列课前准备清单：提示词包发群文件、备用资源（如备用演示对象）、现场设备与网络检查项
- 项目WORKLOG.md追加本次记录

## 禁止

- 不引用具体代码变量名、函数名、类名等实现细节，本skill只描述行为与原则
- 素材不编造：没在项目里发生过的坑、数字、故事一律不写
- 遵循no_ai_style全部规则：
  - 禁自我标榜式措辞（老实说、说实话、坦白讲一类）
  - 禁解释自己用意的句子
  - 禁结尾反问加主动规划下一步（要不要我一类）
  - 禁破折号与英文双引号，引用用「」
  - 禁油腻煽情措辞
  - 禁表态式铺垫与自我宣誓
  - 禁附和措辞（说得准、确实一类）
  - 中文与数字、英文之间不加空格
  - 中文语境禁混英文标点
  - 禁写作手法自评式标签
  - 禁动作隐喻式预告
  - 禁伪情绪反应与戏剧化表演
  - 禁稻草人对比，被否定项必须是读者真实会有的认知
  - 禁营销号告诫与爆款文套话
