# 页型与节奏灵感库（可选）

这是攒下来的页型、互动位置、分钟经验。做课件卡壳时可以翻一翻，但它只是参考，不是模板：可以完全不用，不要照搬，更不要拿它当必须填满的清单。按这门课的内容自己设计，永远优先于这里的任何条目。

## 一、教学序列到页型的映射

不存在标准环节顺序。环节按知识点序列来定：每个知识点配一个微教学循环，循环的每一环挑对应页型。

| 微教学循环的环 | 这一环要干什么 | 可选页型 |
|---|---|---|
| 建立直觉 | 给动机、抛问题、唤起已有经验，让学员想听 | 故事页、大字提问页、对峙比较页、成品live演示页 |
| 讲清 | 把这个知识点拆开讲透 | 链路流程页、结构拆解页、方法卡页、提示词展示页 |
| 演示或例子 | 用真实案例或现场跑给学员看 | 现场演示页、成品live演示页、破案叙事页、数据揭晓页、映射连线页 |
| 检验掌握 | 确认学员真学到了 | 大字提问页（收答案）、站队页（表态）、带做指引页+庆祝检查点页、验收清单页 |

不是每个知识点都要走满四环：多数认知型的点讲清就够，检验交给追问链回扣；要学员亲手做出产出的点才上「建立直觉→讲清→演示→带做检验」全程。难点之间用低负荷的点做缓冲。一个知识点可能横跨多页，也可能一页讲完，由它的难度和产出方式决定。

## 二、带做直播课的节奏参考

只对「讲＋带做」的直播课成立，按需取用，不是硬规则（开场目标感、检验口径、认知顺序这些跨课规则在 SKILL.md 第4步）：

- 方法论显式立卡：全课3到6张方法卡，在对应知识点处立起，收尾合成一页知识点总结。知识点总结永远在最后，方法卡都立完才收得拢
- 互动三件套各就各位：大字提问放在知识点的建立直觉处，站队放在悬念揭晓前，庆祝检查点放在带做完成后

## 三、形态变体

同一套知识点序列，按课堂形态调页型密度，不改知识点顺序：
- 纯讲授：检验环里的「带做指引 + 庆祝检查点」换成「现场演示页 + 提问检验」，总长压到90到120分钟
- 讲 + 带做：核心知识点的检验环用带做，其余用提问或站队
- 工作坊 / 6小时全天：带做密度最高，每个核心知识点都配带做，带做合计可占一半以上时长，各环节按比例放大

## 四、页型目录

每种页型给四件事：用途、构成、动效、讲法要点。讲法要点是排页时定时间和密度用的参考，deck 不内置讲师注记，这些内容不上墙。机制解释类动画不自动播、接翻页键触发（口径见 SKILL.md 5.1），下面动效描述里的循环只适用于氛围和成品演示。页型是积木，按第一节的映射挑用，不绑定固定环节。

1. **封面页**
   用途：定调，交代本课与此前的不同。
   构成：课程名眉题 + 大标题（手动控制断行）+ 一句副题 + 渐变细线 + 底部操作提示。背景放低透明度的主题氛围动效，元素必须与课程主题强相关。
   动效：背景元素慢速循环飘动，装饰线展开。
   讲法：停留一分钟再翻，预告今晚产出。

2. **成品live演示页**
   用途：成品先行，让学员先看到终点。
   构成：左侧两到三条注解（各点一个模块）+ 右侧页内重绘的成品，假数据自动循环刷新。
   动效：数据行滚入、数字跳动、状态脉冲，进入即播离开即停。
   讲法：先让演示自己跑十秒再开口；现场条件允许切真程序，本页是备份。

3. **收获清单页**
   用途：给学员三样带走的承诺。
   构成：编号三条（产出物、方法论、经历），逐条揭示。
   讲法：顺手做课前环境检查（举手统计），别拖到带做环节。

4. **故事页**
   用途：起点叙事。
   构成：纯文字三行递进，讲述腔，零技术词，无图或轻图。
   讲法：放慢，先让设想显得可行，下一页才有落差。

5. **死因页 / 转折页**
   用途：揭示失败与转向。
   构成：大字结论 + 图解动画（如请求被弹回）+ 警示callout。
   动效：把失败过程演出来（请求飞出、被挡、弹回，循环）。
   讲法：重点不是吐槽对方，是抽象出普遍规律。

6. **方法卡页**
   用途：把方法论显式立卡。
   构成：卡名眉题 + 编号三条原则 + 关键句callout。
   动效：逐条揭示，关键句最后出。
   讲法：每条回扣剧情里它出现的瞬间；关键句重读。

7. **大字提问页**
   用途：互动收答案，制造卡点。
   构成：一行大字问题，无其他元素。
   讲法：收2到3个答案全部接住不评判，预算2分钟，别恋战。

8. **对峙比较页**
   用途：两种方案/两种状态的对比。
   构成：左右两panel各带小演示动画，各配一句判词，底部一行结论。
   动效：对比关系空间化成左右对峙，两边动画同时循环。
   讲法：先让学员说出左边的毛病，再给右边。

9. **链路流程页**
   用途：多步骤流程总览。
   构成：横向节点路线 + 每步一个图标 + 人话比喻名 +（括号内技术名）。
   动效：节点逐个亮、连线流动。
   讲法：每步只讲比喻，收尾强调记住有这几步即可。

10. **结构拆解页**
    用途：把一个内部结构讲透（协议包、文件格式一类）。
    构成：分段块 + 每段标注人话含义 + 拆开后的内容物。
    动效：逐段点亮，再演拆开的过程。
    讲法：听个意象即可，预算2分钟封顶。

11. **提示词展示页**
    用途：给学员看可粘贴的真实提示词长什么样。
    构成：对话框样式 + 真实可用的提示词全文 + 盖章动效标注要点（三到四枚）。
    讲法：逐句指读，点出最值钱的那半句；提醒群文件有文本版。

12. **现场演示页**
    用途：讲师现场真跑的衬底。
    构成：重绘的终端或界面窗口，逐行输出动画循环。
    讲法：现场真跑优先，本页动画是备份和讲解道具；关键瞬间停三秒不说话。

13. **带做指引页**
    用途：学员动手环节的任务卡。
    构成：编号三步 + 预期结果（什么现象）+ 卡住三招chips + 底部一行常见情况提示。
    讲法：巡场要点、几分钟时统计进度、翻车预案（环境异常、对象冷清、报错处理）。如果这段带做的重点是展示讲师与AI的协作过程本身（不只是布置任务），优先用23真实过程回放卡页顶替或搭配本页，本页任务卡退化成辅助说明，不是必须的主呈现。

14. **庆祝检查点页**
    用途：带做收口，确认成果。
    构成：大字举手号召 + 一行复盘（刚才走完了什么）。
    讲法：举手统计、拍照发群；没完成的安抚到位。

15. **破案叙事页**
    用途：讲最重的踩坑故事。
    构成：左侧时间线（症状、第一反应、关键动作、结案）+ 右侧证据图表。
    动效：证据图表逐条生长，凶手那条用主题色突出。
    讲法：踩坑故事里分量最重的一个，多给时间；指着图讲证据感。

16. **数据揭晓页**
    用途：实验结果开箱。
    构成：两列大数字 + 柱状图 + 分项结论逐条出 + 反转一行。
    动效：数字滚动、柱子生长。
    讲法：数字滚完再开口；反转（哪些成本不用花）必须讲。

17. **安全红线页**
    用途：凭证、隐私类安全提醒。
    构成：警示callout里逐条红线。
    讲法：放慢语速逐条念，交代它何时会被用到。

18. **映射连线页**
    用途：需求到功能、问题到模块的对应关系。
    构成：左chip → 右chip逐行连线，末行点题（指不回的砍掉）。
    动效：起点chip、箭头生长、终点chip链式出现。
    讲法：可玩反推互动，念左边让学员猜右边。

19. **验收清单页**
    用途：带做成果验收。
    构成：打勾行三条 + 托底一行（没过的Plan B）。
    讲法：逐条举手统计；托底方案念清楚。

20. **知识点总结页**
    用途：全课方法收口。
    构成：编号卡片网格，与课中方法卡一一对应 + 底部一行贯穿口诀。
    动效：卡片交错入场。
    讲法：逐张回扣课中瞬间，提醒拍照，停留给足。

21. **迁移与合规页**
    用途：方法泛化 + 边界声明。
    构成：迁移场景连线两到三行 + 合规warn条（一行内说完）。
    讲法：迁移例子讲具体，合规逐条完整念。

22. **作业与金句页**
    用途：闭环与收尾。
    构成：两条作业（交付物、位置、截止）+ 结尾金句。
    讲法：金句念完就停，不再加话。

23. **真实过程回放卡页**
    用途：把「AI 调试 / 迭代」这类耗时枯燥的真实过程变成可控素材，不用讲师现场干等，也不用静态文字复述一段调试过程。
    构成：一张外链 CTA 卡片（标题 + 一句成果摘要，如「27 条 prompt·4 天·8 章节回放」）+ `target="_blank"` 跳到独立的回放页面，不在 deck 内联嵌入。
    动效：卡片 hover 有反馈即可，不需要复杂动效。
    讲法：回放内容按真实 session 转写提取，不编造对话；提取脚本和渲染页面如果各自维护一份数据（例如脚本写一份 JSON，渲染页面把内容内联进 `<script>` 标签），两者不会自动同步，改完提取逻辑要记得重新生成一次渲染页面用的内联数据，否则页面显示的是旧内容。

24. **五拍分镜页**（可选模式）
    用途：把一个机制或流程按纯分镜讲透，整页几乎不靠文字段落（2026-07-16 Day3以30页130拍整版实证并上线）。
    构成：B0首帧（主体立即可见）+ 5到10个讲师可控拍点（每拍只改画面一处）+ 结论 + 下一问；下一问与下一页标题相接，墙上只留问题、名词、状态、公式。
    动效：拍点全部挂翻页键，至少3拍连续改变同一锚点（同一主体移动、换挡、出结果），讲解顺序由主体的变化承担；未来拍点的内容不得在当前帧可见。
    讲法：每拍讲一句，拍完即翻；适合机制链路、网络路径这类有空间感的知识。

## 五、时间预算参考

本节只供用户自己排时长参考，AI不主动引用（红线5）。

- 3小时晚场：净讲162分钟左右 + 休息10到15分钟。带做合计约90分钟（占一半），其余按知识点难度分配，难点和带做多给、缓冲点少给
- 6小时全天：讲授演示90到100分钟 + 带做150分钟以上 + 休息若干，收尾知识点总结不超过10分钟
- 90分钟纯讲授大致量级：动机开场6 + 每个核心知识点10到15 + 收尾6，按知识点数量摊进净时长
- 单页停留经验值：大字页1到2分钟、内容页2到3分钟、破案页5分钟、带做页按预期完成时间给

## 六、文案模式

- 标题钩子只用于建立直觉环的页面（故事、开场、悬念页）：把抽象教学点翻成具体场景句（动作加对象）。讲清环的知识页标题直接用知识名词，不翻场景句；疑问句只用在提问页和站队页（硬约束版在teaching-voice上墙形态）
- 关键句：一页一句，单独成行，是讲者要重读的那句话。callout样式承载
- 比喻选材：从学员日常生活里选（快递、电话、查户口、记账本），一个比喻服务一个概念，全课比喻总数控制在十个以内，重要比喻要在后文回扣
- 金句收尾：与本课方法强绑定的一句话，必须从课程内容里推得出来，不写放之四海皆准的口号
- 带做提示词：写成学员可直接粘贴的完整指令，包含目标、格式、约束（先跑通最简版本一类），不含技术术语
