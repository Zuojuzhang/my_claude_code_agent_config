# 视觉体系（页型、风格适配、视觉收口）

第5步做视觉前读。UI按页型落地，不按单页美化：先把页面按教学功能归类，再给每类页型定一套可复用视觉系统。只把几个显眼页面单独做漂亮，整份课件会变成风格拼贴。

## 页型清单

常见页型至少包括：

- **封面 / 章节转折页**：负责建立题目和节奏。首屏要有项目或主题的第一视觉信号；章节转折页要让学员知道「现在在问什么、下一段回答什么」。
- **问题页 / Q页**：负责追问链转场。统一成一套可识别的问题系统，例如编号、阶段标签、主问题、下一步提示。主问题可以强，但辅助信息要轻，避免像普通大标题页。
- **截图 / 产品成果页**：负责展示真实产出。统一用截图框或设备框承载，包含轻量标题栏、窗口点、来源或状态标签；不同项目按领域适配外观，例如数据产品像仪表盘窗口，移动产品像手机框，文档产品像页面预览。不要让同一份deck里一会儿裸图、一会儿卡片、一会儿浏览器框。
- **技术动画 / 机制解释页**：负责把抽象机制讲成人能看懂的流动过程。统一放进「演示舞台」：浅色或暗色画布、稳定边框、标题栏、统一阴影、统一节点和箭头语言。动画内部内容随主题变化，但舞台容器要一眼看出这是机制演示。机制动画不自动播：接到翻页键节奏上，讲师按「下一步」先把当前动画完整播一遍，再继续揭示或翻页；只有氛围装饰和成品live演示允许自动循环。
- **步骤 / 流程页**：负责让学员不迷路。用步骤轨、流程轨或面包屑回扣全貌，轨道存在感要弱于当前页标题和主体。
- **总结 / 作业页**：负责收束和行动。总结页把方法抽象出来，作业页展示交付标准和真实成果示例，不要只放一句口号。

## 概念视觉身份

连续机制页不能按页重新设计。开工前先做概念分级和视觉词典，确认学员到底需要记住什么：

- **必须记住**：课程主干角色或对象。给固定颜色、固定形状、固定位置和固定名字，跨页不变。
- **只需理解**：连接关系、动作和状态。用线、箭头、小票、结果回执、遮罩或高亮表达，不升级成新角色卡。
- **可以略读**：协议名、方法名、字段名。作为二级标签贴在对应关系附近，不与主概念争抢视觉层级。

视觉词典至少锁定四个维度：

| 维度 | 表达什么 | 规则 |
|---|---|---|
| 颜色 | 概念身份 | 同一概念跨页不换色；强调只改变饱和度、透明度或描边强度 |
| 形状 | 对象类型 | 角色、软件、协议、消息、结果、说明书各用固定形状；协议、方法、消息不画成人物卡 |
| 位置 | 关系方向 | 指挥方、执行方、外部软件和回传方向尽量固定；不要每页重排一次总图 |
| 动画 | 状态变化 | 只表示出现、选中、移动、调用、回传和完成，不用动画改变概念身份 |

连续页优先复用一张持久总图，只做高亮、遮罩、局部放大、消息移动和结果回传。每页最多引入1个新术语，同时高亮的主实体不超过3个。若概念跨多页，先做2张代表页样板：一张检查首帧是否能看懂，一张检查完整揭示态是否仍然干净；用户确认后再铺开。

## 风格适配五步

UI体系要抽象成「结构角色」，不是抽象成某个deck的具体颜色或文案。迁移到新课件时按顺序适配：

1. 先确定主题气质：数据看板、工具软件、内容创作、业务分析、生活助手、硬件设备，还是别的类型。
2. 再定一组项目主色和辅助色，主色只用于识别和强调，不要让整份deck变成单色主题。
3. 把截图框、问题页、机制舞台、步骤轨都换成同一套主色语言，但保留各自的信息层级。
4. 根据素材类型换外壳：网页截图用浏览器窗，移动截图用手机框，命令行用终端窗，文档或报告用纸张/报告壳，流程机制用演示舞台。
5. 对已有deck做视觉升级时，优先统一页型和容器，不先改文案；文案改动会牵动课程主线，除非用户明确要求。

## 视觉收口清单

一份deck最后做一次视觉收口，逐项检查：

- **进度轨弱化**：章节轨、问题轨、面包屑是导航提示，不是主视觉。完成态用低饱和灰或淡色，当前态用细边框或浅底强调即可，避免实心高饱和按钮抢标题。
- **截图框统一**：同类截图用同一框架和同一阴影等级；框内图片不再额外套不同圆角和阴影。截图框标题栏文字短，说明性文字放旁边或下方。
- **技术动画统一**：所有机制动画页共用同一演示舞台外壳。节点、连线、标签、提示气泡的圆角、边框和阴影统一；只有内容和颜色强调随知识点变化。
- **问题页成体系**：Q页不散落成不同版式的大字页。编号、阶段标签、主问题、下一步提示统一；是否显示大号水印编号按风格决定，同一deck内保持一致。
- **首屏非空**：每页主视觉和核心内容立即可见，frag只给次要remark做逐步揭示；新页做完核对首屏不是空的。
- **可访问性**：中文断行、字号、对比度、reduced-motion没有明显问题。
- **响应式收口**：固定舞台deck按舞台设计尺寸验，并换2到3个差别大的窗口尺寸和比例（含非16:9）确认每页渲染完全一致；旧的非固定舞台deck至少用一个标准宽屏和一个窄预览宽度验关键页（截图页、技术动画页、四列卡片页、总结页）。
