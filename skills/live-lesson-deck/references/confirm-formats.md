# 确认格式（第3步知识点清单、第4步页序大纲）

发确认前读本文件。格式固定，不自选形式。

## 备课文档排布

- 标题、来源和课程定位摘要放在最前面，后面立即放`## 最新修订记录（最新在前）`。
- 每条修订用`### YYYY-MM-DD 修订主题`作为锚点，按时间倒序排列。同一天有多条时，后发生的放前面。
- 新修订插在修订区第一条之前，旧记录保留；不要追加到文件末尾，否则用户每次都要翻到底部才能看到当前决策。
- 知识点清单、稳定基线、历史页序和废止方案放在修订记录之后。

## 知识点清单确认

1. 先给困惑链逐环视图，这是主体产出物。每环一行三段：环号、学员的问题（用学员的话，真实会问出口的那句）、一句话回答。用户逐环检查问题冒不冒得出来、回答顺不顺，例：

   ```
   1. LLM为什么只会说不会干活？ → 它本质是预测模型，训练时只学了接话
   2. 那想让它干活怎么办？ → 干活的本质是运行电脑命令，得有东西替它执行
   3. 谁替它执行？ → Harness在你本地跑命令，充当它的手脚
   ```

2. 再给挂环知识点表，每个板块一张表，五列：知识点、挂链上哪一环（环号）、所属知识域、学完达到什么状态、来源。编号跨板块全局连续，方便用户按编号提修改。来源只允许四种标注：项目事实、用户口述、AI补充已核、AI补充未核。挂不上环的知识点不进表，进待确认项。
3. 表格后给「不教清单」：知识域完整性判断（SKILL第3步第4条）里决定不教的部分，每条一句为什么不教（超出定位、学员用不上、留给后续课）。取舍本身要摆出来让用户确认，不许默默跳过。
4. 倒推视图（能力块 ← 缺什么知识）降级为可选：链和挂环表已覆盖依赖关系时不发，用户点名要备料视角再发。
5. 最后固定跟「待确认项」，集中列AI拿不准的假设。
6. 固定问法收尾：直接回复确认，或按环号、编号说改哪条。

## 页序大纲确认

- 开头一行总量：共几页、几章、预计几分钟。
- 按章节分组，每章标页码范围和预计分钟数，章内表格四列：页码、页标题、这页回答的问题、页型与素材需求。「这页回答的问题」写上一页冒出的那个问题，追问链断没断，用户扫一遍就能看出来。
- 表格后列「素材缺口」：哪几页缺真实截图、logo、回放素材。这块是知会不是门槛，默认用占位继续做，占位页交付时集中列入待确认项。
- 固定问法收尾：直接回复确认，或按页码说改哪页。

## mini页序大纲（修改已有deck的大改用）

只列受影响区段，格式同页序大纲，落盘到该deck的plan文档，插入最新修订区顶部，不新开文件。
