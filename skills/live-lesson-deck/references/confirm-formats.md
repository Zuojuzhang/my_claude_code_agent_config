# 确认格式（第3步知识点清单、第4步页序大纲）

发确认前读本文件。格式固定，不自选形式。

## 知识点清单确认

1. 先给倒推视图，每个能力块一条链，让用户一眼看到「要做成这件事到底缺哪些知识、知识之间怎么关联」，例：

   ```
   数据获取 ← 缺：爬虫是怎么回事 ← 缺：HTTP轮询和WebSocket的区别
   数据存储 ← 缺：数据库最小知识（建表、写入、查询）
   看板呈现 ← 缺：图表选型 ← 缺：实时刷新机制
   ```

2. 再给分板块表格，每个板块一张表，四列：知识点、学完达到什么状态、挂在主线哪一环、来源。编号跨板块全局连续，方便用户按编号提修改。来源只允许四种标注：项目事实、用户口述、AI补充已核、AI补充未核。
3. 表格后固定跟「待确认项」，集中列AI拿不准的假设。
4. 固定问法收尾：直接回复确认，或按编号说改哪条。

## 页序大纲确认

- 开头一行总量：共几页、几章、预计几分钟。
- 按章节分组，每章标页码范围和预计分钟数，章内表格四列：页码、页标题、这页回答的问题、页型与素材需求。「这页回答的问题」写上一页冒出的那个问题，追问链断没断，用户扫一遍就能看出来。
- 表格后列「素材缺口」：哪几页缺真实截图、logo、回放素材。这块是知会不是门槛，默认用占位继续做，占位页交付时集中列入待确认项。
- 固定问法收尾：直接回复确认，或按页码说改哪页。

## mini页序大纲（修改已有deck的大改用）

只列受影响区段，格式同页序大纲，落盘到该deck的plan文档，追加「修订记录」，不新开文件。
