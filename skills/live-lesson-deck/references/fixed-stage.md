# 固定舞台（防止 deck 随窗口重排）

## 要解决的问题

deck 若用视口相对单位（vw/vh、clamp 里的 vw）做字号和留白，布局就跟着窗口的尺寸和比例实时变。预览面板、投影仪、验证脚本各是一个窗口，同一页在这个尺寸挤满或溢出、在另一个尺寸正常。作者按一个尺寸验证，别的尺寸的问题抓不到，用户却在自己的窗口里撞见。「测更矮/更多尺寸」是治标，窗口尺寸无穷多，测不完。

固定舞台把这条路堵死：内容在一个固定画布里排一次，整体等比缩放铺满窗口，任何窗口只是放大缩小，所见即所得。验一个尺寸等于验所有尺寸。

## 怎么做

1. 一个固定尺寸的舞台容器（如 1920×1080），deck 所有页放在里面，容器本身尺寸写死。
2. JS 按窗口算等比缩放系数，用 transform:scale 铺满，不足比例居中留边；resize 时重算：
   - `scale = min(窗口宽 / 舞台宽, 窗口高 / 舞台高)`
   - 舞台绝对定位居中，`transform: translate(-50%,-50%) scale(scale)`。
3. 舞台内所有尺寸用相对舞台的单位，二选一：
   - 直接用 px（按舞台设计尺寸给定值）。舞台缩放时 px 跟着缩，稳。
   - 把舞台设成容器（`container-type: size`），字号留白用 cqw/cqh（1cqw = 舞台宽的 1%）。缩放对它们同样有效。
   - 禁止 vw/vh 和 clamp(...vw...)：它们相对视口不相对舞台，缩放对它们无效，会破坏固定舞台。

## 从已有的响应式 deck retrofit

若 deck 已经用 vw/vh 铺开（历史遗留），改造顺序：
1. 加舞台容器 + 缩放 JS，先把整体框起来。
2. 把内容尺寸单位从 vw/vh 换成 px 或 cqw/cqh。clamp(min, Xvw, max) 里的 Xvw 换成相对舞台的等价值（在设计尺寸下 Xvw 折算成 px 或 Xcqw）。
3. 逐页在多个窗口尺寸下比对，直到每页渲染一致。

## 验证（替代「测很多尺寸」）

固定舞台做对了，就不必挨个尺寸测排版，改成测一致性：
- 在 2 到 3 个差别大的窗口尺寸和比例下（含非 16:9）加载，逐页量 `.wrap` 子元素 boundingRect 相对舞台的占比，确认每页相对布局完全一致。
- 一致，说明舞台真的固定；不一致，说明还有内容用了视口相对单位，回去改。
