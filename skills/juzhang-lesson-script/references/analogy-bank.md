# 类比库（全部verbatim原句，按类比域分类）

> 用法：写稿需要类比时，先查本库有没有现成资产（同概念直接复用原话术）；没有的按「类比工序」造新的，新类比的取材域按优先级选。
> 硬规则：类比是一次性消耗品，同一知识点复讲时裁掉比喻、改挂术语；金句和案例话术才是背熟资产，逐字复用。

## 类比工序（六条，写稿直接执行）

1. 类比物优先级：**此刻现场 > 学员职业母语（数据分析）> 职场关系 > 日常生活物件**。绝不用学员没摸过的东西打比方。
2. 铺垫三板斧：「举个例子啊」「你想想」「我给大家说一个神奇的事情啊」，或先演反面场景。
3. 讲完必带确认句，承重类比加拆边界或标不确定度（「大概可以这样理解」）。
4. 收回必降压：「仅此而已」「就这么简单」「知道有这么回事就行了」。
5. 拟人糙话（偷看、PUA、牛马、套娃、陨落、刺客）不铺垫不解释，贴完就走。
6. 重要类比可以立了再拆：先让大家认可，再主动推翻或标注适用边界，落到更准的结论。

立了再拆的完整范例（L1主agent领导类比）：「理论上来说，我的主 agent 就像你是一个团队一样，你不可能这个领导自己去把所有事情都做了吧？」→ 让大家扣一认可 → 「为什么大家会觉得没问题呢？……因为这非常符合人类的直觉。……所以我说人类其实非常自恋」→ 收回到「只要你是信息的传递，一定会有缺失的」。

## 一、职场上下级域（最大的类比仓库）
