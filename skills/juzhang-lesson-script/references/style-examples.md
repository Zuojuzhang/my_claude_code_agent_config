## 5. 讲解流水线五变体

- **方法论课**（福格模型、反漏斗）：现象悬念 → 开放提问 → 接答案 → 指出共同缺口 → 引入框架 → 拆要素（每要素配大白话）→ 反直觉判断 → 跨域例子巩固 → 回本案例 → 泛化到学员生活。
- **概念课**（HTTP、WebSocket、Git、localhost、端口）：探底投票 → 最小模型或对比 → 真实环境实测（F12、任务管理器）→ 提炼本质（重要句说两遍）→ 推广到生活例子 → 划边界 → 确认。
- **实操验收场**（L3全课）：抛问题逼直觉（明确允许错）→ 复读接住每条弹幕 → 给自己的第一判断（措辞刻意强硬）→ 反问链推理 → 跨场景类比加固 → 收束成一条原则 → 承认教学案例的妥协不装完美。
- **弹幕插播**（缓存命中）：挂起降压 → 择机回收 → 从学员已有经验搭场景 → 落定义 → 说清对用户的实际意义 → 再降负担 → 挂新闻八卦做记忆钩子 → 吐槽收尾。
- **决策类**（模型选型、云数据库）：抛问题制造预测 → 替学员说出直觉答案 → 翻转 → 给判断框架（两三个维度）→ 实测数据压实 → 拉到商业后果 → 给出本项目决策并留退路（「如果说它后面有问题，我们可以再换」）。

## 6. AI初稿 vs 橘长改稿（修改对比，diff-001加diff-004，最重要的一节）

> 学的是修改方向，不是终态。每对先看差异说明，负例只为对照，正例才是模板。后续每份修改对（diff-002起）的精华条目继续追加到这里。

**AI写的**：现在这些大厂，正在排着队把自己的数据开放给AI用
**橘长改的**：现在这些大厂，都在把自己的数据开放给AI用
差异：拟态修饰（排着队）删掉，事实平说。他的口语是平的，密度来自信息不来自劲

**AI写的**：先说爬的这个。错误的。这个我们在第一幕砍需求的时候其实已经砍过一刀了
**橘长改的**：先说爬虫这个方案，是绝对错误的。这个我们需求模块的时候，其实就已经否决了
差异：「错误的。」独立判决降为句内判决（预写稿没有现场答案可判）；「第一幕」这种制课内部词换成学员视角的「需求模块」

**AI写的**：大家都是分析师，这个逻辑大家秒懂啊。你写报告的时候
**橘长改的**：大家都是分析师对吧，就像你写报告的时候
差异：不替学员宣称理解状态，用确认句问；补了类比引导词「就像」

**AI写的**：AI也一样，让它背事实，它就开始编，让它解释事实，它才值钱。就这么简单。
**橘长改的**：AI也一样
差异：硬造的对仗金句整句裁掉。金句只能从背熟资产库取或从内容里长出来，不靠句式对称撑

**AI写的**：我们实测过，你直接问AI，我楼下有什么餐厅，它是真敢答。【补真实案例：AI编店名的实测对话或截图】
**橘长改的**：你可以试试直接问AI，新江湾字节楼下有什么餐厅，它会告诉你有莫尔顿牛排、有西塔老太太烤肉，还有费大厨。但我告诉你，这三家，字节的楼下一家都没有，字节楼下有什么？有星巴克、有墨西哥餐厅，但字节人一般都不吃这些餐厅，因为字节自己就有食堂
差异：占位符被亲自填充，结构是可复现邀请（你可以试试）→ 具体地点 → AI的具体错误输出 → 翻转 → 在地生活细节收尾；「我们实测过」改成学员可复现的「你可以试试」

**AI写的**：这里还能回扣一下第二节课的东西啊。大家发现没有
**橘长改的**：但大家发现没有
差异：结构动作直接做，不预告「回扣」

**AI写的**：凭的是你叠在上面的私有记忆。你的六个本子，你不吃香菜……这句话大家记一下啊，外部数据人人能查，叠上私有记忆，才是你的产品（重复2遍）
**橘长改的**：对于这个产品来说，你的私有化就是用户的私有记忆，其实就是上下文，你不吃香菜……外部数据人人能查，结合上下文才是你的产品（1遍）
差异：收口挂回课程主线概念（上下文），不像人话的内部命名（六个本子）弃用；金句说一遍，「大家记一下」和预排重复删掉

**AI写的**：好，那接上高德是不是就万事大吉了？也不是。有一个边界要给大家说清楚
**橘长改的**：Ok，那接上高德是不是就万事大吉了？也不完全是，还记得我们的产品规划吗？有一个功能要做过敏原检测
差异：抽象的「边界」落到产品规划里的具体功能，判断语气软半档（也不完全是）

**AI写的**：合出来的结果，我自己看了都觉得意外。／还有一个更让我意外的。
**橘长改的**：合并出来的结果，这太离谱了。／还有一个最离谱。。
差异：「意外」是AI的平板情绪词，他的情绪词是离谱，配情绪标点「。。」。修省略不能只补语法，词要换成他的词库

**AI写的**：为什么呢？材料太少，它抓不到你稳定的模式，只能拿它自己那套腔调来凑数。
**橘长改的**：为什么呢？因为材料太少，它抓不到你稳定的模式，只能拿它自己那套恶心的AI腔调来凑数。
差异：自问的答句补「因为」（连接词必须写出来）；主动加态度词（恶心的），态度是真实判断不是修辞

**AI写的**：这个活大家熟悉吗？这就是你们天天做的用户行为分析，对不对？不看用户嘴上说什么，看行为日志。我嘴上说我没口头禅，日志里是772个「对吧」。
**橘长改的**：这个分析方法你们应该都很熟的，对吧？这就是你们天天做的用户行为分析，不看用户嘴上说什么，看他做了什么
差异：指代具体化（这个活→这个分析方法）；书面词换大白话（行为日志→他做了什么）；机灵回环句（我嘴上说……日志里是772个）整句删，回环呼应是卡兹克的机制不是他的

**AI写的**：只做一件事：把原句摘出来。
**橘长改的**：让他们把原句摘出来。
差异：悬念前置框（只做一件事：）删掉，有话直说

**AI写的**：说白了，拿证据说话，不拿印象说话。
**橘长改的**：（整句删）
差异：对称格言收口不写。被删的不是「说白了」（他的真词），是后面那个对称句

**AI写的**：仅此而已，这个事没有门槛，只有信息差。
**橘长改的**：仅此而已，这件事最大的门槛是上下文，如果你没有积累自己的上下文，也就没法蒸馏自己，你也要意识到，从现在开始，一定要积累了
差异：首尾修辞闭环换成真判断加行动号召；他的长句一逗到底连着流，「短句才像口语」是AI的误解

## 7. 场景变体要点

- **讲授场vs复讲**：理论只讲授一遍，第二遍压成清单（skill五件套从3000字讲授压到150字验收清单再压到一行速记）；复讲裁比喻改挂术语；案例和金句逐字复现，金句还会升格（「skill不是写出来的，是调出来的」从段落收束升到全课压轴加「打包票」背书）。
- **搭档课**：用问句递话筒（「那这里就请你来给大家讲一讲」），对方讲完由他升维翻译成概念（「你相当于做了一个中介」）；接话先回音再展开。
- **纯弹幕课**：互动全靠复读接答；冷场处理（「我掉了吗？OK，吓我一跳，大家都不说话。」）；收尾闲聊更长，暴论浓度更高。
- **演示课**：第二人称指路一步一动作（「来，你右上角有一个三个小点，点击设置」）；操作前给防坑提示加理由（「不要点进这里边，因为你要找的是 API 啊」）；等AI跑任务时转答疑或预告等待时长；倾向不直播干等（「我直接调好做成一个回放给大家看就行了」）。
