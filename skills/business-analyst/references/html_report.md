# HTML长页报告规范

阅读型分析报告的默认形态。单文件HTML，滚动阅读，高信息密度，交付即可在浏览器打开。
本文档管三件事：动手前的设计判读、页面的版式与密度规则、交付前的质量闸。
2026-07-11定稿，源于同一份报告五个版本的实测返工教训。

---

## 一、动手前：设计判读（跳过即返工）

写任何代码前，先输出一行设计判读：

> 读者是〈谁〉，场合是〈什么〉，视觉方向承诺为〈一个明确的方向〉。

规则：

1. **每次现场承诺方向，禁止吃老本。**上一份报告的成品配色字体不是这一份的默认。
   方向可以是编辑刊物、暗色科技、运动杂志、极简画廊等任何家族，但必须说得出为什么配这个读者。
2. **读取使用者的审美偏好存档**（memory或项目CLAUDE.md），作为边界约束。
   偏好存档约束的是边界（要什么感觉、拒绝什么），不是具体色值字体。
3. **本机若装有taste-skill**（`~/.claude/skills/taste-skill`），动手前读其主SKILL.md，
   其禁令与本文档冲突时以更严者为准。
4. **信息密度是硬约束不是风格选项。**分析报告不做「一屏一句话」的deck式呈现，
   全部章节内容（假设检验、口径、图表、解法、局限性）必须完整装载。

## 二、版式骨架

结构自上而下，章节内容由阶段一锁定的分析模板决定：

- **进度条**：CSS scroll-driven animation实现（`animation-timeline: scroll()`加
  `@supports`守卫）。禁止`window.addEventListener('scroll')`。
- **粘性导航**：单行，锚点直达各章，当前章高亮用IntersectionObserver。
- **Hero**：文本元素最多4个（眉标、主标题、一句副题、关键数字带）。主标题手动控制断行，
  副题一句话说完。关键数字带算数据资产不算文本。
- **正文章节**：每章开篇一条粗规则线加结论式标题。布局家族至少4种轮换
  （编号叠排、双栏不对称、小倍数图组、不等宽bento、满幅金句），同一家族不连用三次。
- **原声引用**：分析对象是文本时（用户提问、反馈、评论），至少2处匿名化原文引用，
  做成独立的排版时刻。数据里有人，报告里要看得见人。
- **附录**：局限性清单加机器可复核的存档路径（原始数据、分类标签、统计口径）。

## 三、图表语法

- **标题写结论句**，不写维度名。「问的是职业，不是技术」是标题，「主分类占比」是图注。
- **一图一强调**：承载结论的mark用强调色，其余一律中性灰阶。全页只有一个强调色。
- 值标在mark端部，只标关键值；细线基线；图表不垫底色卡片。
- 数字用等宽或表格数字对齐；文字永远用文本色，不穿数据色。
- 风险标注（【小样本】【推断】【数据不足】）视觉上可辨识，跟着图注走。
- 条形生长、渐入等动效走IntersectionObserver触发，全部受`prefers-reduced-motion`约束。

## 四、质量闸（交付前逐项勾选，任何一项不过就回去改）

版式与主题：

- [ ] 整页锁一个主题（亮或暗），章节不中途反色；同色系深浅一档的带状区允许
- [ ] 全页一个强调色，出现处全部落在结论上
- [ ] 圆角制度全页统一（全直角、全圆角或有成文规则）
- [ ] 眉标（小号大写宽字距标签）每3章至多1个
- [ ] 中点`·`每行至多1个，不做万能分隔符
- [ ] 破折号零出现（中文用逗号句号，区间用「至」或连字符）
- [ ] 跑马灯至多1处
- [ ] 等宽三卡禁用，多卡用不等宽组合
- [ ] 布局家族≥4种，同一家族不连用三次

密度与内容：

- [ ] 每屏至少一个完整信息单元（图表加解读、完整表格或一组结论），无一句话大字页
- [ ] 分析模板的公共锚点章节全部在场：商业假设、Actionable Insight、分析局限性、References
- [ ] 每张图表标题是结论句且一图一强调
- [ ] 文本类分析含≥2处匿名化原文引用
- [ ] 所有数字有口径（时间范围、过滤条件、全量或去重）
- [ ] 风险标注不缺位

工程底线：

- [ ] 无`window.addEventListener('scroll')`，滚动效果走CSS scroll-timeline或IntersectionObserver
- [ ] `prefers-reduced-motion`下全部动效退化为静态
- [ ] 移动端`document.documentElement.scrollWidth`等于视口宽（严查行内样式压住媒体查询）
- [ ] 中文标题无词中断行（用手动`<br>`加字号验算控制）
- [ ] 正文对比度WCAG AA以上，正文行长32至42字
- [ ] 字体走font-display swap，CJK有回退栈

## 五、渲染自检（质量闸的执行方式）

用无头浏览器跑两轮截图，逐屏人工看过才算完成：

1. 桌面1440宽，按视口高度分屏截到页尾，逐屏检查断行、溢出、对比度、图表标签。
2. 移动390宽，首屏加中段各一张，读`document.documentElement.scrollWidth`，
   大于视口宽即横向溢出，用逐元素`getBoundingClientRect`定位（排除overflow hidden容器内的元素）。
3. 检查中发现的每处问题，修完重截，直到两轮零缺陷。
