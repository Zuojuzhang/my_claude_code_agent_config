# 定位报告成品审校方法

## 一、目标

审校对象是已经生成的 HTML、JSON、Markdown 三件套，不重新做定位分析。审校必须回答三个问题：内容有没有误导，图表有没有隐藏信息，屏幕和打印交付是否一致。

基础审校必须运行：

```bash
python3 scripts/review-report.py positioning-report
```

来源时效警告允许报告通过，但必须保留在结果中；只有用户明确要求零警告交付时才使用 `--strict`。不得通过虚构发布日期、删除不利来源或改高置信度消除警告。

## 二、内容语义门禁

- HTML、JSON、Markdown 的报告标题和推荐定位必须逐字一致。
- JSON 保留机器枚举，HTML 与 Markdown 必须显示中文标签。
- 有证据编号的单元格不能再出现“证据不足”前缀。
- 空否决项显示“暂无否决项”；缺少优势证据才显示“证据不足”。
- 个人 IP 的商业价格没有参与定位比较时显示“未纳入定位比较”；历史人物或无价格场景显示“不适用”，不能显示“不适用（未核验）”。
- `execute`、`validate_first` 和 `insufficient_evidence` 必须继续受证据门槛约束，不能为了报告好看升级状态。
- 事实、推断、假设和建议不得混写；动态来源警告必须可见。

## 三、图表完整性门禁

- 1 至 5 级只能描述为规则化诊断，不能写成统计概率或市场份额。
- 散点图出现相同坐标时，保留真实数值，只做确定性像素偏移；气泡内显示编号，图下索引完整显示名称，图注说明“同分点轻微错位”，表格和 tooltip 保留精确值。不得依赖 `hideOverlap` 隐藏名称。
- 雷达图固定使用五个有业务含义且有证据的数据维度，不形成综合人物能力总分。竞品雷达第五维“心智显著”必须来自对应对象的心智阶梯等级；缺少该证据的对象不进入雷达。
- 雷达系列名称使用图外完整索引，不能在移动端折叠成 `1/3` 分页图例或裁掉名称。
- 没有数据时显示空态，不创建默认柱形、默认中间值或假装完整的雷达轮廓。
- 网页与打印均检查图例；不能只看到 `1/3` 而遗漏其余系列。
- 图表使用静态确定性渲染，避免打印或截图在动画中间捕获。

## 四、屏幕与移动端门禁

至少检查 `1440×1000`、`390×844` 和 `375×812`：

- 顶部导航吸附、压缩和当前章节高亮正常。
- 点击每个章节后，标题不会被吸顶导航遮挡。
- 页面本身没有横向溢出。
- 长表允许横向滚动，并在右侧仍有内容时显示非文字边缘提示；滚动到末端后提示消失。
- 文字、标签、图例和状态徽标不互相覆盖。
- 三张散点图的索引数量与数据点数量一致；三张雷达图各有五个维度，系列索引数量与实际系列数量一致。
- 复杂表格允许横向阅读，不通过缩小到不可读字体强行塞满一屏。

## 五、打印与 PDF 门禁

打印审校不能只检查浏览器中的 `@media print` DOM。浏览器截图正确，不代表分页 PDF 内部 SVG 尺寸正确。

1. A4 页边距为左右各 17mm，正文固定为 176mm。
2. 进入打印媒体后重算 ECharts 容器尺寸。
3. 热力图和方案、优势雷达图在打印时使用单列全宽，确保轴名和全部图例可见。
4. 生成 PDF 后必须栅格化抽检：首页、需求散点、竞品图、优势图、方案图、行动表、验证图和证据附录。
5. 检查 SVG 是否空白、裁切、跨面板串位或覆盖后续表格。
6. 长表跨页时表头应重复，章节标题不能孤悬在上一页底部。

## 六、自修复顺序

1. 先修错误语义和数据映射。
2. 再修图表遮挡、同分点和打印尺寸。
3. 再修移动端表格提示与阅读密度。
4. 最后做字体、间距、斑马纹和装饰性细节。

不得用删数据、缩小到不可读、隐藏来源或降低证据警告的方式修复视觉问题。
