# 输入采集与就绪门槛

定位研究不能从“帮我做个定位”直接跳到竞品检索。先接受用户最方便的资料形式，再把输入标准化，最后通过就绪门槛。门槛未通过时只补信息，不展开正式研究，也不提前生成差异化结论。

## 一、支持的输入方式

三种方式可以单独使用，也可以混合使用：

1. **引导问答**：用户直接回答问题。适合资料尚未整理、希望边聊边澄清的情况。
2. **附件资料**：读取用户明确上传或指定的 PDF、Word、PPT、表格、Markdown、文本、图片及其他可解析文件。适合已有公司介绍、产品资料、客户访谈、销售记录、案例或课程材料的情况。
3. **官网与 URL**：打开用户提供的官网、产品页、价格页、案例页、公开数据库或其他具体网址，并继续追溯原始页面。适合需要核验当前公开表达和市场事实的情况。

混合输入优先级最高。例如：官网用于理解公开定位，产品文档用于理解交付，访谈附件用于理解用户，公开来源用于研究竞品。

安全边界：只读取用户明确提供的文件和网址，不扫描私人目录、浏览记录、邮箱或内部系统。链接打不开、附件损坏或内容无法提取时，必须说明失败项并请求替代材料。

## 二、统一输入记录

无论用户采用哪种形式，都先整理为符合 `templates/intake-brief.schema.json` 的 `intake-brief.json`：

- `subject`：名称、类型、身份或业务主体。
- `objective`：这次定位要支持什么决策。
- `offer`：实际提供什么产品、服务、课程或价值。
- `market`：地域、语言、行业和时间范围。
- `target_user_hypothesis`：当前认为谁最需要，允许是待验证假设。
- `inputs`：文本、附件和 URL 清单，以及是否可访问。
- `known_competitors`：用户已知竞品或经常被比较的对象，允许为空。
- `constraints`：不能承诺、不能服务、不能进入或必须遵守的边界。
- `unknowns`：用户不知道或尚未验证的信息。

输入不是证据等级。用户回答通常先标记为 `self_report`；公司官网通常证明“公司如何表述自己”，不能单独证明效果、用户认知或领先地位；内部材料必须核对作者、日期、样本和适用范围。

## 三、提问方式

每轮只问 1 至 3 个会阻塞下一步的问题。每个问题必须包含四项：

1. **问题**：只问一个意思。
2. **为什么问**：说明它会改变哪项判断。
3. **示例**：给出 2 至 3 个具体但不诱导的答案。
4. **允许不知道**：告诉用户不知道时可以提供什么替代资料。

禁止一次抛出十几个抽象问题，也禁止用术语要求用户先理解“品类阶梯、心智占位或 D6”。

## 四、问题库与示例

### 1. 定位对象

- 问题：这次要定位的是谁或什么？
- 为什么问：决定分析对象、竞争集合和报告结构。
- 示例：`一家企业服务公司`、`一门面向管理者的 AI 课程`、`我的个人专家品牌`。
- 不知道时：提供官网、公司介绍或产品资料，由 Agent 先识别。

### 2. 定位目的

- 问题：你希望这次定位解决哪个具体决策？
- 为什么问：获客、品牌升级、推出新品和内部统一会产生不同结论。
- 示例：`让官网更容易被客户理解`、`进入中国大陆企业市场`、`从个人顾问转成课程品牌`。
- 不知道时：说明当前最困扰的一件事，例如“客户总把我们和代运营混在一起”。

### 3. 实际提供的产品或服务

- 问题：客户最终购买和得到的是什么？
- 为什么问：定位不能脱离真实产品和交付能力。
- 示例：`季度 GEO 运营服务`、`6 周线上训练营`、`企业级监测软件加顾问服务`。
- 不知道时：上传合同、报价单、产品页或交付清单。

### 4. 目标市场

- 问题：本次优先分析哪个地域、语言和行业？
- 为什么问：同一产品在不同市场面对的竞品、法规和用户认知不同。
- 示例：`中国大陆中文市场，消费品牌`、`新加坡英语市场，B2B SaaS`、`全国市场，暂不限行业`。
- 不知道时：至少给出当前主要客户所在地域。

### 5. 目标用户

- 问题：谁会决定购买，通常在什么情况下开始寻找方案？
- 为什么问：使用者、决策者和付款者可能不同，场景决定竞争对象。
- 示例：`品牌负责人发现 AI 回答品牌信息错误时`、`创业者准备第一次系统获客时`、`HR 要改善团队下午状态时`。
- 不知道时：提供最近 3 至 5 个客户、销售线索或咨询问题，允许匿名。

### 6. 可证明的能力

- 问题：哪些事实可以证明你能做到，而不只是你认为自己擅长？
- 为什么问：没有证据的能力不能成为核心差异。
- 示例：`可复核的客户案例`、`专利或资质`、`持续交付数据`、`独有流程和样本`。
- 不知道时：上传案例、客户反馈、交付记录或方法文档；没有材料时如实写“尚未验证”。

### 7. 已知竞争对象

- 问题：客户最常把你和谁、哪种方案或“不做”进行比较？
- 为什么问：定位针对真实替代选择，而不是只看同类公司。
- 示例：`两家 GEO 服务商`、`客户内部自建团队`、`继续做传统 SEO`、`暂不投入`。
- 不知道时：可以留空，由 Agent 根据目标用户和场景建立候选竞争集合。

### 8. 约束与取舍

- 问题：有哪些事情不能承诺、不能做或不愿做？
- 为什么问：《定位》要求聚焦和牺牲，边界也影响可信度。
- 示例：`不保证排名`、`不服务医疗行业`、`不做低价标准化课程`、`必须符合中国广告法`。
- 不知道时：提供合同边界、合规要求或当前拒单标准。

## 五、研究就绪门槛

只有以下六项全部具备，才开始外部研究和竞品对比：

1. 定位对象可识别：名称和类型明确。
2. 决策目的明确：知道这次研究要帮助做什么决定。
3. 产品或服务可描述：至少能用一句话说清客户买到什么。
4. 市场边界明确：至少有地域和语言，行业可暂定。
5. 目标用户有初步假设：至少知道可能的决策者或触发场景。
6. 至少有一项可检查的主体资料：用户文本、附件或官方 URL 任一即可。

运行 `python3 scripts/validate-intake.py intake-brief.json` 执行机器校验。未通过时，状态为 `intake_incomplete`。只输出校验器返回的缺失项和下一轮最多三个带示例的问题，不做竞品研究，不生成定位方案。

## 六、诊断与推荐门槛

研究就绪只允许开始采集，不等于可以正式推荐。完成研究后继续检查：

### 可以形成完整诊断

- 当前公开表达至少有一项第一方资料。
- 核心直接竞品有原始或官方来源；标准模式通常为 3 至 5 个，不足时必须解释。
- 至少有一类用户侧或真实业务侧资料，用于判断需求和选择标准。
- 至少有一项能力、案例或交付资料，用于判断优势是否可证明。
- 地域、时间窗口、来源日期和未覆盖范围已记录。

未满足时只能输出“初步观察 + 补证计划”，不能形成完整差异化诊断。

### 可以给出正式推荐

- 推荐定位同时有需求、竞争和能力证据。
- 核心竞品至少有官方来源，且至少一个关键竞品有用户侧证据。
- 关键市场或趋势主张使用 A/B 级或有公开方法的来源。
- 关键效果主张有独立验证或可复核数据，不能只依赖自述和单个成功案例。
- 重要冲突、反证条件和剩余未知项已公开。

证据方向成立但未达正式执行门槛时使用 `validate_first`；核心链条缺失时使用 `insufficient_evidence`。不得为了输出完整报告而自动补齐结论。

## 七、来源相关性门

每条关键证据同时通过六项检查：

1. **来源权威性**：来源是否有资格陈述该事实。
2. **主张适配性**：来源是否真的能证明当前主张。
3. **直接性**：是原始事实、二手转述还是推测。
4. **时效性**：是否仍适用于当前市场。
5. **独立性**：是否来自不同原始生产者或数据过程。
6. **市场适配性**：地域、语言、行业、客群和样本是否与本项目匹配。

其中任何一项明显不匹配，证据必须降级或排除。来源等级高不代表与当前问题相关，例如美国市场的高质量报告不能直接证明中国大陆客户的购买偏好。

## 八、用户可见的进度说明

开始研究前，用简短清单告诉用户：

- 已收到哪些输入形式。
- 当前如何理解定位对象和目的。
- 研究就绪门槛是否通过。
- 哪些内容是用户自述，哪些将外部核验。
- 下一步将研究哪些用户、竞品和市场事实。

这样用户可以在研究成本发生前纠正对象、目的或范围。
