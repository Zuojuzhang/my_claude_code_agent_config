# 定位理论体系

本文件把《定位》的核心思想转化为可执行判断规则。它是战略方法，不是市场事实来源；具体项目仍需用户、竞品和市场证据。

## 一、核心命题

### 1. 定位发生在心智中

- 定位对象是潜在用户如何理解、归类和选择主体。组织自我描述作为研究输入。
- 功能差异只有被用户感知、记住并用于比较时，才形成定位差异。
- 报告必须区分“产品事实”和“用户心智位置”。

### 2. 用户处于信息过载环境

- 用户会用简单品类、关键词、代表对象和既有经验降低决策成本。
- 长清单、复杂概念和多重承诺通常削弱记忆。
- 定位表达应聚焦一个主要品类和一个核心差异，不把全部优势塞进一句话。

### 3. 心智按品类形成阶梯

- 用户通常先理解“这是什么”，再判断“为什么选它”。
- 同一品类中的领导者、跟随者和新进入者面临不同策略。
- 新进入者不能假设自己可以用更多功能直接替代已经占据心智的位置。

### 4. 定位是相对于竞争参照成立的

- 没有“用户会拿谁来比较”，就没有可验证的差异化。
- 竞争参照不仅是同类产品，也包括间接方案、现状、不行动和心智标杆。
- 竞品研究优先回答它占据了什么认知，不只列功能。

### 5. 领导者、跟随者与新类别

- 若目标品类已有稳定代表对象，新进入者应寻找可防守的细分位置、重新定义竞争参照或创建用户能理解的新类别。
- “创建新品类”必须同时满足：用户需求真实、品类可理解、自身能交付、教育成本可接受。
- 未发现竞品不能自动推导出新品类机会。

### 6. 聚焦意味着取舍

- 清晰定位必然排除一部分用户、场景、功能或表达。
- 不说明“不服务谁”“不解决什么”，通常意味着定位仍未完成。
- 技术上能做的事情，不等于战略上应该进入定位的事情。

### 7. 名称和关键词是心智入口

- 名称应帮助用户快速判断品类或差异，避免只有内部人员能理解的造词。
- 心智关键词用于压缩定位，不承担完整论证。
- 命名建议必须经过无提示理解测试，不凭作者喜好确定。

### 8. 再定位先处理旧认知

- 已有品牌或个人 IP 再定位时，必须识别旧认知、旧用户、旧渠道和资产延续关系。
- 新定位如果与旧认知直接冲突，需要更高教育成本和更强证据。
- 报告必须写清“从什么旧认知转向什么新认知”。

### 9. 延伸会稀释位置

- 同一名字承载过多不相关品类，会降低品类清晰度和信任转移效率。
- Skill 不把“可以卖更多产品”自动解释为“应该共享同一定位”。
- 若新业务与现有心智距离过大，应比较子品牌、新品类或独立产品路径。

### 10. 定位需要长期一致执行

- 定位不止是一句口号，还要映射到产品、内容、价格、渠道、销售和交付。
- 表达与体验不一致时，体验会覆盖表达。
- 推荐定位必须附验证路径和执行取舍，不能停在文案层。

## 二、战略角色、竞争动作与常见陷阱

### 1. 先判断自己在阶梯中的角色

- **领导者**：重点是维护品类代表性、持续强化原有位置，并用新品牌或独立路径处理距离过大的新业务，不能假设领导地位会自动延伸。
- **跟随者**：不能复刻领导者表达，应寻找用户在意且领导者不便占据的相反特征、细分场景、价格带或交付方式。
- **新进入者**：优先寻找未被清晰占据的可理解位置；“第一个进入心智”比“第一个做出功能”更重要，但必须由用户认知证据验证。
- **新品类创建者**：先建立品类参照和教育路径，再争取代表位置；名称、机制和核心价值必须共同降低理解成本。

### 2. 对手再定位

- 当目标心智位置已被强势占据时，不用更大的自我声明硬碰，而是指出现有参照在特定用户、场景或决策标准上的不适配。
- 再定位只能基于可核事实和清晰边界，不能歪曲、贬损或把“未发现公开证据”写成对手缺陷。
- 报告必须说明要改变的旧认知、替代的新参照、教育成本和可能反击。

### 3. 名称陷阱

- 名称是品类与差异进入心智的入口，含义模糊、只对内部有意义或与既有品类冲突的名称会增加解释成本。
- 缩写、技术代号、过度抽象造词和无法承载新位置的旧名称都应进入理解测试。
- 好名称不能补救错误定位；命名必须在品类、用户、竞争参照确定后进行。

### 4. 搭便车与延伸陷阱

- 不能因为现有品牌、流量或知名度可用，就默认所有新产品共享同一名称与定位。
- 把同一名字延伸到相距较远的品类，可能短期降低获客成本，却长期稀释原有位置。
- 评估延伸时比较：用户是否自然联想、信任能否迁移、旧位置是否受损、独立品牌的成本是否更低。
- 延伸并非一律否决。只有当新产品仍强化同一品类位置、用户自然接受、名称不会产生错误假设且旧位置不被稀释时，才进入可行方案；报告必须同时比较独立品牌路径。

### 5. 传播一致性

- 定位需要足够长时间和足够一致的产品、内容、渠道、价格与销售表达才能进入心智。
- 频繁更换核心品类、关键词或竞争参照，会让每次传播从零开始。
- 创意只能放大定位，不能替代定位；所有传播创意必须回到同一核心认知。

## 三、定位六问诊断门

形成推荐方案前逐项回答：

1. **现在占据什么位置？** 用用户认知和比较证据回答，不用内部愿景代替。
2. **希望占据什么位置？** 位置必须单一、具体、可理解，并说明不服务谁。
3. **必须替代谁或什么旧认知？** 明确直接竞品、间接方案、现状或不行动。
4. **资源是否足以进入并长期守住？** 检查产品、证据、预算、渠道、时间和组织一致性。
5. **能否长期坚持？** 检查业务扩张、内容节奏与短期增长压力是否会破坏聚焦。
6. **表达与位置是否一致？** 名称、品类、价格、产品体验、内容和销售是否共同强化同一认知。

任一关键问题无法回答时，推荐状态不得直接进入 `execute`。

## 五、统一定位判断链

按以下顺序执行，不得从自我优势直接跳到口号：

1. 明确定位对象、身份、阶段和商业目标。
2. 确定最佳目标用户与关键选择场景。
3. 建立真实竞争集合，识别用户当前参照。
4. 描述各参照已占品类、关键词、价值和信任证据。
5. 识别用户重要需求、触发、决策标准和现状不满。
6. 验证自身资产是否具备用户相关、稀缺和可证明性。
7. 形成 2 至 3 套品类与差异组合。
8. 比较品类清晰度、用户价值、竞争独特性、执行成本和风险。
9. 推荐一套，并说明牺牲项、旧认知和反证条件。
10. 用用户理解、差异回忆、偏好、试点或转化验证。

## 六、定位表达结构

推荐结构：

> 为【最佳目标用户】，在【关键场景】下，提供【品类】，通过【独特机制】实现【核心价值】；相较于【主要替代方案】，关键差异是【可证明优势】。

该结构是决策摘要，不替代证据链。每个字段必须能映射到报告数据：

| 字段 | 必须回答 | 最低证据 |
|---|---|---|
| 最佳目标用户 | 谁最在意该价值 | 用户访谈、行为或购买证据 |
| 关键场景 | 何时产生选择 | 任务路径、触发事件 |
| 品类 | 用户把它理解为什么 | 无提示理解、搜索与比较行为 |
| 独特机制 | 为什么能交付 | 产品、方法、流程或资源证据 |
| 核心价值 | 用户获得什么结果 | 需求与结果证据 |
| 主要替代方案 | 用户当前拿谁比较 | 竞品、现状或不行动证据 |
| 可证明优势 | 为什么更适合 | 相对基线与独立证据 |

## 七、必须避免的理论误用

- 把“第一”理解为宣传词，而不是有边界和证据的心智位置。
- 为了成为新品类第一而创造用户无法理解的概念。
- 只研究同类产品，忽略现状、不行动与替代方案。
- 把“独特”与“重要”混为一谈。
- 用内部产品功能替代用户选择逻辑。
- 用一次搜索结果判断心智地位。
- 将定位简化为 slogan、标题或视觉风格。
- 用绝对总分掩盖证据不足和维度冲突。
