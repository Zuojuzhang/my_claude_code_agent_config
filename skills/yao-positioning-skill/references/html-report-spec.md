# HTML 定位报告设计规范

## 一、视觉方向

Kami 纯白编辑部式研究报告：页面、正文和模块使用纯白背景，依靠冷灰细线与极轻阴影建立层级；使用墨蓝主色、衬线标题、无衬线正文、等宽数字与低饱和绿/金/红语义色。首屏直接交付人物或对象、推荐定位、推荐状态和关键可信度，不使用营销式 Hero、渐变、玻璃效果或装饰性图表。

设计目标：

- 管理层能在首屏判断“现在被理解成什么、要占什么词、与对手哪里不同”。
- 分析人员能从结论下钻到竞品、证据和来源。
- 观察数据、诊断分值、假设和风险具有不同视觉语义。
- 375px 移动端、桌面端和打印版均可读。

## 二、信息架构

- 桌面端：顶部双层吸附导航，第一层提供报告身份与打印动作，第二层提供横向章节导航；下滑后压缩高度并保持当前章节高亮。
- 移动端：同一顶部导航横向滚动，自动把当前章节移到可视区域；正文单栏。
- 首屏顺序：报告对象 → 当前认知 → 目标心智词 → 一句话定位 → 三位主要对手 → 可信理由与战略牺牲 → 推荐状态和下一步 → 六项核心指标。
- 首屏只读取 `positioning_snapshot` 与推荐方案一致字段；完整推理、反证和来源放在折叠详情及后续章节。
- 章节：定位、可信度、需求与机会、竞争与心智、优势与课程四层、方案、落地建议、验证、证据附录。

## 三、视觉编码

| 数据类型 | 视觉规则 | 含义 |
|---|---|---|
| 真实观测 | 实色、百分比或真实数值 | 直接来自来源、样本或记录 |
| AI 诊断 | 纹理或 1 至 5 级 | 按规则形成的分析判断 |
| 假设 | 低饱和琥珀提示 | 需要后续研究验证 |
| 风险 | 克制红色与反证条件 | 可能推翻推荐 |
| 缺失数据 | 灰色空态与文字 | 不得用模拟值填图 |

## 四、图表清单

| 章节 | 决策问题 | 图表 |
|---|---|---|
| 研究可信度 | 来源和主张有多可靠 | 来源等级柱图、主张置信度图 |
| 用户需求 | 哪些需求重要且供给不足 | 重要性 × 不满程度气泡图 |
| 竞争格局 | 用户在比较哪些选择、哪些认知已被占据 | 竞争角色覆盖图、编号索引式竞品定位散点图、五维竞品定位画像雷达、心智阶梯图 |
| 优势诊断 | 哪些优势具备相对价值 | D6 优势热力图、核心优势结构雷达 |
| 定位方案 | 哪套方案更值得选择 | 品类清晰度 × 竞争独特性气泡图、方案权衡雷达 |
| 市场验证 | 什么结果支持继续 | 验证门槛条形图与实验表 |

图表旁保留一句解释和原始表格，不能只有图没有结论。

散点图出现相同坐标时，必须保留原始 1 至 5 级数值，仅用确定性的像素偏移分开标记，并在图注明确“同分点轻微错位，精确值见表格”。标签应缩短显示，完整名称与原值保留在 tooltip 和原始表格中。

竞品表提供竞争类型分段筛选；证据附录提供主张类型筛选。图表 tooltip 支持鼠标和点击，显示判断理由、置信度与证据编号。证据编号可跳转到证据台账，来源编号可继续跳转到来源附录。

## 五、技术约束

- 使用本地 `assets/echarts.min.js`，优先 SVG 渲染。
- HTML 不请求 CDN、字体、分析脚本或远程资源。
- 图表容器使用稳定高度，避免加载后布局跳动。
- Tooltip 使用 `richText`，不执行用户 HTML。
- 数据由内嵌 `application/json` 提供，`<`、`>`、`&` 必须转义。
- 用户文本使用 `textContent`，不能拼接为 `innerHTML`。
- 本地文件只显示文件名，不暴露绝对路径。

## 六、可访问性

- 第一焦点项为跳到正文链接。
- 导航使用 `<a>`，动作使用 `<button>`。
- 图表容器提供 `role="img"` 和中文 `aria-label`。
- 所有交互目标不小于 40 × 40px。
- 提供可见键盘焦点。
- 遵循 `prefers-reduced-motion`。
- 不能只用颜色传递状态，状态同时显示文字。

## 七、响应式与打印

- 1280px：顶部吸附导航与两列图表布局。
- 840px 以下：导航继续横向滚动，主内容与复杂信息模块单栏化。
- 640px 以下：指标两列、图表单列、表格横向滚动；未滚动到末端时用克制的右侧内阴影提示剩余列，到达末端后提示消失。
- 375px：无横向页面溢出，长中文标签完整显示。
- 打印：隐藏导航和工具，A4 正文固定为扣除页边距后的 176mm；复杂热力图与雷达图改为单列全宽，图表在打印介质切换时按真实容器重算；图表与表格避免跨页切断，链接不显示交互色。

## 八、设计禁区

- 不使用紫蓝渐变、玻璃卡片、圆形光斑或营销式大标题。
- 不把所有章节包装为相同卡片。
- 不使用超过 8px 的卡片圆角。
- 不用饼图展示五类以上数据。
- 不将诊断分值伪装成客观统计。
- 不生成缺失数据图表。
