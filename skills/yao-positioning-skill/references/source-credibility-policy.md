# 来源可信度与证据政策

来源等级只回答“来源本身通常多可靠”，不能回答“它是否适合证明当前主张”。每条关键结论必须同时检查来源质量、主张适配、直接性、时效性和独立性。

## 一、来源等级

| 等级 | 典型来源 | 可用于 | 主要限制 |
|---|---|---|---|
| A | 原始著作、政府与监管数据、官方产品与价格页、原始论文 | 理论原意、当前价格、功能、日期、官方主张 | 官方营销结果仍需外部验证 |
| B | 专业协会、有公开方法的行业研究、同行评审综述、可核查数据库 | 市场结构、趋势和行业数据 | 样本、地域、口径和时间可能不匹配 |
| C | 可追溯访谈、认证购买评价、公开社区、应用商店评论、项目方材料 | 用户认知、痛点、异议、自述与行为线索 | 单样本不能外推，需要聚合和去偏 |
| D | 署名媒体、专业分析、垂直行业文章 | 背景解释、线索和交叉验证 | 不能替代原始来源 |
| E | 聚合页、转载、SEO 内容、匿名帖、AI 摘要、搜索摘要 | 发现线索 | 不得支撑关键结论 |

## 二、按主张匹配来源

| 主张 | 优先来源 | 不能单独依赖 |
|---|---|---|
| 当前价格、功能、套餐 | 官方购买页、产品文档、应用商店 | 旧评测、媒体转载 |
| 竞品如何定位自己 | 官网首页、产品页、创始人公开表述 | 第三方一句话介绍 |
| 用户痛点与认知 | 访谈、评论、搜索意图、销售或流失记录 | 品牌官网案例 |
| 市场规模与趋势 | 政府、协会、有方法说明的研究 | 无方法的市场文章 |
| 产品效果 | 独立验证、可核数据、完整案例 | 自述、单一成功故事 |
| 心智位置 | 重复用户比较、搜索和认知证据 | 官网关键词频次 |
| 定位理论 | 原著、作者或出版社材料 | 二手摘要 |

## 三、事实标签

- `fact`：可由外部资料直接核实。
- `self_report`：用户、竞品或组织自己的陈述。
- `inference`：由多条证据推导出的判断。
- `hypothesis`：需要后续验证的临时解释。
- `recommendation`：基于事实和推断形成的行动建议。

报告不得把自述升级为事实，也不得隐藏推断步骤。

## 四、时效窗口

默认窗口，项目可按行业调整：

| 信息 | 默认窗口 | 超期处理 |
|---|---|---|
| 价格、功能、人员、运营状态 | 90 天 | 标记需复核，不进入当前事实图 |
| 首页定位、内容主题、渠道表达 | 180 天 | 仅用于历史对比 |
| 市场统计和行业趋势 | 优先 24 个月 | 说明年份、口径和局限 |
| 理论原典 | 不设普通时限 | 核对作者、版本和原始出处 |

动态资料记录 `published_at` 或 `accessed_at`。不知道日期时降低置信度。

## 五、独立性与同源处理

- 同一新闻稿、报告、数据集或品牌稿件的转载共享一个 `origin_group`。
- 两个独立域名不等于两个独立来源。
- 独立佐证必须来自不同原始生产者或不同数据过程。
- 品牌官网与创始人采访通常属于同一组织视角。
- 多条评论需要检查平台、时间、样本和重复发布。

## 六、关键结论格式

每个关键判断使用：

> 结论 → 支撑证据 → 推理过程 → 置信度 → 冲突证据 → 反证条件

置信度只使用：

- `high`：直接、当前、两个以上独立来源，无重大冲突。
- `medium`：一个强来源或多个部分支持来源，需要继续验证。
- `low`：间接、过期、小样本或存在明显冲突，只作研究线索。
- `insufficient`：没有适配来源或无法确认原始出处，不得形成确定结论。

置信度是规则化判断，不是统计概率。

## 七、证据台账字段

使用 `templates/evidence-ledger.md`，并确保 JSON 至少包含：

- `claim_id`、`claim_text`、`claim_type`；证据对象用 `claim_ids[]` 回链一个或多个主张。
- `source_id`、标题、发布者、链接或文件、等级和用途。
- `perspective`：`official`、`user`、`independent`、`internal` 或 `mixed`，用于稳定区分来源视角；不能从任意 `source_type` 文本猜测。
- 发布日期或访问日期、直接性和 `origin_group`。
- `evidence_id`、摘要、相关性、直接性、时效性和独立性。
- `market_fit`：`aligned`、`partial`、`mismatch` 或 `unknown`；`market_fit_notes` 说明地域、语言、行业、客群和样本口径。
- 冲突证据、置信度、反证条件和报告使用位置。

## 八、研究安全

- 只访问完成任务需要的公开页面或用户明确提供的文件。
- 不隐式扫描私人目录、浏览历史、邮箱或内部系统。
- 搜索摘要只用于导航，必须打开原始页面。
- 报告 HTML 只允许 `http` 和 `https` 来源链接；本地路径仅显示文件名。
- 用户输入在 HTML 中必须通过 `textContent` 或转义输出，不能执行为代码。

## 九、相关性与输入类型

高等级来源不一定适合当前项目。每条关键证据必须同时检查：来源权威性、主张适配性、直接性、时效性、独立性和市场适配性。市场适配至少覆盖地域、语言、行业、客群和样本口径；任一关键项明显不匹配时降级或排除。

| 输入 | 默认解释 | 可以证明 | 不能单独证明 |
|---|---|---|---|
| 用户问答 | `self_report` 或内部线索 | 主体目标、约束、经历和待验证假设 | 市场需求、领先地位、用户认知和效果 |
| 用户附件 | 依据作者与形成过程标记为内部、官方或用户侧 | 文档中直接记录的产品、流程、案例和样本 | 超出样本与时间范围的普遍结论 |
| 官方官网或产品页 | 官方第一方来源 | 公司如何表述自己、当前公开功能、价格与案例声明 | 用户真实认知、实际效果和竞争领先 |
| 用户提供的第三方 URL | 按原始发布者和方法评级 | 与其研究方法、样本和范围匹配的主张 | 未披露方法或与本市场不匹配的结论 |
| 搜索结果与 AI 摘要 | E 级导航线索 | 帮助发现原始页面 | 任何关键结论 |

对中国大陆市场的推荐，不能仅用境外样本推导购买偏好、监管结论或竞品格局。境外资料可用于方法和趋势背景，但必须明确地域局限，并寻找本地证据交叉验证。

`market_fit=aligned` 表示关键适用范围与目标市场一致；`partial` 表示可用于部分背景或机制；`mismatch` 表示不能用于当前主张；`unknown` 表示适用范围尚未确认。`execute` 推荐至少要引用一条 `aligned` 证据，且不能用高等级但不适配的来源绕过该门槛。
