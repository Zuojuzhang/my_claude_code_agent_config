00 / DEFINITION

一句话理解这套系统

定位 Skill 是一个以竞争参照和证据为约束的战略诊断系统。它先确认信息是否足够，再研究用户与竞争，随后形成定位选项、推荐结论和验证计划。

INPUT

接收多种材料

引导问答、公司介绍、附件、官网链接、竞品名单、访谈纪要与已有研究可混合输入。

SYSTEM

执行受控判断

通过信息门槛、可信度分级、竞品比较、D6 优势诊断和方案压力测试控制结论质量。

OUTPUT

交付可行动结论

输出一句话定位、差异化解释、证据账本、风险边界、行动优先级与市场验证路线。

POSITIONING FORMULA

为【最佳目标用户】，在【关键场景】下，提供【品类】，通过【独特机制】实现【核心价值】；相较于【主要替代方案】，关键差异是【可证明优势】。

01 / PRINCIPLES

原理与理论体系

系统以《定位》的心智竞争观为主轴，引入现代定位方法、课程营销结构和证据治理，形成一条从市场事实到战略建议的判断链。

P01 / MIND

心智先行

定位的判断对象是目标用户如何理解与记住主体。企业自我描述只能作为输入，市场认知决定真实起点。

P02 / CATEGORY

品类与阶梯

先识别用户使用的品类语言，再判断主体位于哪一级心智阶梯，以及新的空位是否存在。

P03 / COMPETITION

竞争参照

差异化必须绑定替代方案。竞品集合覆盖直接、间接、现状、不行动和标杆五种参照。

P04 / FOCUS

聚焦一个位置

系统压缩受众、场景、品类和价值表达，让一句话能够进入心智，并约束产品与传播的一致性。

P05 / EVIDENCE

证据约束

事实、自述、推断、假设和建议分层记录。关键主张同时接受权威性、直接性、时效性与独立性检查。

P06 / CONSISTENCY

长期一致

定位会进入产品、销售、内容和组织决策。持续使用同一套品类与价值语言，才能形成认知积累。

THEORY CORE 心智中的相对位置

围绕用户、场景、品类、竞争和证据，寻找一个清晰、相关、可信、可守的认知位置。

《定位》心智、品类、阶梯、聚焦、重新定位、品牌延伸风险
现代定位竞争替代、独特属性、价值映射、最佳匹配用户、市场类别
课程营销人群、痛点、结果、机制四层表达，连接价值与购买理由
证据认识论事实分级、来源可信度、推断边界、可证伪假设、验证计划
02 / ARCHITECTURE

六层系统架构

架构把触发、输入、研究、判断、验证和呈现分离。JSON 数据合同贯穿各层，HTML 与 Markdown 从同一份结构化数据生成。

L01

触发与路由层

显式 Skill 调用、自然语言意图识别、范围判断

输出：任务类型与研究模式
L02

输入与门槛层

问答、附件、网址、混合材料；最少必要信息校验

输出：intake-brief.json
L03

研究与证据层

市场、用户、竞品、品类与来源可信度管理

输出：研究账本与证据卡
L04

诊断与决策层

需求分析、D6 优势诊断、定位选项、压力测试

输出：推荐结论与风险
L05

合同与验证层

Schema、数据校验、证据引用、内容一致性检查

输出：可信报告数据
L06

呈现与治理层

离线 HTML、Markdown、视觉 QA、安全与分发

输出：可交付报告包

03 / WORKFLOW

AI 运行流程

完整流程包含 14 个步骤，并归并为五个决策阶段。每个阶段都有明确输入、产物和继续执行条件。

PHASE 01

识别任务

  1. 识别定位对象
  2. 明确决策目的
  3. 选择研究深度
PHASE 02

建立输入

  1. 整理问答与材料
  2. 标准化输入 brief
  3. 通过准备度门槛
PHASE 03

研究市场

  1. 验证主体事实
  2. 建立竞品集合
  3. 识别需求与品类
PHASE 04

形成判断

  1. 诊断 D6 优势
  2. 生成 2 至 3 个选项
  3. 压力测试并推荐
PHASE 05

交付验证

  1. 生成报告数据
  2. 渲染 HTML 与 Markdown
  3. 执行 26 项 QA

04 / FEATURES

系统特点

这些特点共同解决三个核心风险：材料不完整、差异缺少竞争参照、建议缺乏证据和验证路径。

01

多模态输入

支持逐题问答、附件、官网、具体网址和混合材料；输入统一转换为标准 brief。

02

准备度优先

信息通过最少必要门槛后进入正式研究，减少早期臆测带来的错误方向。

03

竞品驱动

先建立替代方案全集，再比较差异强度、用户相关性和证据强度。

04

D6 优势诊断

从相关、稀缺、证据、防守、战略和表达六个角度评估优势质量。

05

证据感知

每条重要主张带来源、时间、市场适配、可信度和事实类型标签。

06

多方案比较

形成 2 至 3 个战略选项，用统一维度比较收益、风险和验证成本。

07

通俗表达

详细分析前先给出当前理解、竞品理解、差异标签和一句话推荐。

08

可审计交付

HTML、JSON 与 Markdown 保持同源，报告结论可回查到证据账本。

05 / BOUNDARY

能力边界

边界用于保护结论可信度，也帮助团队把定位判断、专项研究、传播创意和经营决策分配给合适的角色。

适用范围

  • 个人、产品、服务与公司品牌定位
  • 竞品与替代方案研究
  • 用户需求和差异化诊断
  • 定位选项比较与推荐
  • 定位验证与行动路线

外部协同

  • 完整市场规模测算与投资模型
  • 法律、财务、医疗等专业结论
  • 单独的广告语与创意生产
  • 品牌知识库的长期运营
  • 投放、销售和渠道的执行管理

人类决策保留

  • 战略野心与资源承诺
  • 价值观、伦理与声誉取舍
  • 组织能力建设优先级
  • 预算与时间窗口选择
  • 管理层最终共识与责任归属
安全与隐私边界

系统不需要密钥即可生成离线报告；远程脚本默认关闭；输入路径执行清理；HTML 内容执行转义；研究材料按必要范围使用。涉及机密材料时，应继续采用本地研究模式，并由用户确认可引用范围。

06 / SCENARIOS

个人与公司使用场景

同一套判断框架可以服务个人和组织。两类场景共享心智、竞争、聚焦与证据原则，研究对象和行动约束各有侧重。

对个人

PERSONAL POSITIONING
专家与顾问

确定服务对象、专业品类、关键问题、独特方法和可信证明。

内容创作者

连接身份标签、受众认知、内容母题与商业化路径。

职业转型者

识别可迁移优势、目标岗位参照和进入新心智阶梯的策略。

课程主理人

提炼目标人群、关键痛点、结果承诺、独特机制和证据。

对公司

CORPORATE POSITIONING
公司品牌

统一公司是谁、服务谁、解决什么、凭什么相信与如何长期占位。

产品与服务

明确最佳匹配客户、竞争替代、价值差异和销售表达。

新品类进入

评估新类别名称、教育成本、心智空位和资源匹配。

重新定位

识别旧认知、目标认知、竞争反应和迁移路径。

01 / 诊断当前市场如何理解主体
02 / 选择比较不同定位方向
03 / 对齐统一产品、销售与内容语言
04 / 验证用市场反馈更新判断

CLAIM CONTROL

可信来源还需匹配具体主张

来源等级只解决一部分问题。系统会继续检查这条材料是否直接支持当前结论、是否属于目标市场、是否足够新，以及是否存在独立交叉验证。

  • 关键事实至少对应一个直接来源。
  • 高风险建议需要独立或一手证据支持。
  • 推断和假设必须显式标记，并进入验证计划。

09 / EVOLUTION

进一步价值与演进方向

当定位报告进入真实经营流程，它会逐步形成战略共识、证据记忆和市场学习机制。以下能力值得在后续版本继续强化。

F01 / ALIGNMENT

决策共识层

把管理层分歧显式化，记录各选项的假设、代价、责任人与否决条件，提升战略对齐效率。

F02 / FALSIFICATION

反证机制

主动寻找会推翻推荐定位的证据，为每个核心假设设置失败信号和停止条件。

F03 / PORTFOLIO

品牌组合护栏

分析公司品牌、产品品牌和个人 IP 的关系，控制品牌延伸造成的认知稀释。

F04 / EXPERIMENT

定位实验系统

把一句话定位转化为访谈、落地页、销售话术和内容测试，用市场数据逐步校准。

F05 / MEMORY

组织证据记忆

保存历史结论、版本变化、被否决选项和新证据，避免团队重复研究同一问题。

F06 / CADENCE

定位运营节奏

建立季度复盘、竞争信号监测与年度重评，让定位随市场事实更新，同时保持核心一致。
