# 搭建决策导向的 BI 看板

## 这个 skill 在治什么病

最常见的看板失败不是图不好看，是**数据陈列**：把每个字段都画成一个图摆上去，能算的指标都塞进 KPI 卡，看着满满当当，但没有一个具体的人能拿它做任何决策。

根因是两个问题从来没被回答：**谁在看？他看完要做什么决定？** 一旦这两个问题悬空，选图就退化成「这个字段画什么图好看」，而不是「这个判断需要哪张图」。

好看板的判断力不在文字解说里，在**选了什么图、怎么切分维度、用什么统计量、怎么排序**里。数据自己会说话，前提是你替读者把问题问对了。

下面六步是通用流程，任何领域的看板都适用。

## 核心原则（先记住，再看步骤）

1. **受众和问题先于图表**：没有具体受众和决策，不要开始画。
2. **一个模块回答一个具体问题**：图表跟着问题走，不跟着字段走。
3. **让图表自己说话**：判断藏在选型和切分里，不写结论旁白、不写「本看板回答XX」、不写「视角是XX」。
4. **选对统计量**：均值、总量这类聚合最容易骗人，按数据分布选中位数/分位数/门槛值。
5. **诚实标边界**：数据答不了的问题，宁可声明，不用不严谨的方法硬造。
6. **先验证再画**：统计结论先用代码或手算确认站得住，再写进图。

## 工作流

### 第一步：摸数据（动手画之前必做）

先把数据的底摸清楚，否则后面所有判断都是空中楼阁：

- **结构**：有哪些字段、什么粒度（一行代表什么）、多少行、覆盖什么范围。
- **量级与分布**：每个数值列的最小/最大/中位/分位。重点看**是不是长尾或有头部**，这决定了后面该用均值还是中位数。
- **缺失与脏值**：哪些列有空值、异常占位（比如某条记录某字段被抓成 0 或空串）、量纲错误。**脏值要在数据加载层兜住**（兜成合理默认或显式标记），别让它污染图表和统计。
- **能答什么、不能答什么**：把数据真正支撑的结论粒度想清楚。没有某个维度的字段，就答不了关于那个维度的问题，这一步就要识别出来，留到第五步如实声明。

产出：你能一句话说清这份数据是什么、能支撑到什么程度。

### 第二步：锁定受众和他的决策（地基，跳过必返工）

这一步是整个看板的地基。用一句话写出来：

> **谁在看这个看板？他带着什么决策来？**

然后把他要做的**3 到 6 个具体决策**列出来，每个决策对应后面一个看板模块。决策要具体到能落地，不是「了解情况」这种虚的。举例（仅示意通用形态，不限领域）：

- 「该把目标定在多少」「现在该不该入场」「该对标谁」「该追哪个方向」，这些是决策。
- 「看看播放量」「了解UP主分布」，这些不是决策，是字段，是病。

**两个常见卡点**：

- **受众候选不止一个**（比如既可以服务一线执行者，也可以服务管理者，两者要的图完全不同）：先给出你推荐的受众和理由，再跟用户确认。这是会改变整个看板的地基决策，值得一问，不要替用户默认。
- **数据撑不住这个受众**：如果选定的受众最关心的问题，数据根本答不了（缺关键字段），要么换一个数据能服务的受众，要么明确告诉用户「要服务这个受众得先补什么数据」。不要为了迁就受众去硬画答不了的图。

### 第三步：一个模块回答一个问题（选型跟着问题走）

每个模块开工前先问：**这块要回答受众的哪个决策问题？** 再根据问题的类型选图，而不是根据字段选图。常见映射：

| 受众的问题形态 | 选什么图 |
|---|---|
| 某指标随时间怎么变（在涨还是跌、拐点在哪） | 折线 / 面积趋势图 |
| 群体怎么构成、是否集中、可不可复制 | 分布图（直方图 / 分桶堆叠条） |
| 谁强谁弱、该对标谁 | 排名条形榜（必要时按维度切换） |
| 两个指标什么关系、要不要追某个指标 | 散点图 / 四象限（按中位线划分） |
| 关键水位 / 门槛是多少 | 单值 KPI 卡 |
| 要查具体某条记录 | 可排序、可筛选、可搜索的明细表 |

几条选型纪律：

- **KPI 卡只放与决策相关的数**，不堆指标。一张卡放一个读者能直接对照自己情况的标尺，比五张「总量」卡有用。
- **筛选和切分维度本身就是分析**。给趋势图加一个「门槛线 / 平均值」切换、给排名榜加一个「按次数 / 按均值」切换，等于让读者自己问出更细的问题。哪些维度值得切，取决于受众的决策。
- **明细表是样本库**，定位成「按维度检索具体对象」，配排序、筛选、搜索，不是把所有行铺出来。

### 第四步：选对统计量（别让指标骗人）

选哪个统计量，本身就是分析判断，不是默认 average 了事：

- **有长尾或头部时，均值会骗人**。少数爆款会把平均值拉到一个没人达得到的虚高位置，吓退或误导读者。改用**中位数、分位数、或门槛值**（比如「及格线」「最低准入值」），给读者一个能对照自己的真实标尺。
- **总量对个体决策往往没用**。「累计总额 XX 亿」这种大数好看但不可行动。换成个体能对照的单位量（人均、单条、每期）。
- **比例和率比绝对值更能比较**。不同体量的对象放在一起比时，先归一化。
- 选完统计量，**先用代码或手算验证结论站得住**，再写进图。避免画完才发现数错、或洞察根本不成立。

### 第五步：让图表自己说话（不写文字旁白）

这是区分「专业看板」和「带旁白的数据陈列」的关键。看板要靠图表传达判断，不靠文字解说。

**删掉这些**（它们是在替图表念结论，或暴露元信息）：

- 结论句 / 解读条（「XX 说明 YY」「这意味着 ZZ」「先想清楚你要……」）。
- 「本看板回答以下问题」「视角是 XX 用户」这类自我说明。
- 模块标题里的「回答：XXX？」式问句标签。
- 卡片、图例下面那行价值判断式的小字。

**判断改为内嵌在结构里**：你选了中位数而不是均值、你按某条线划了四象限、你筛了高频对象、你用某个维度排序，这些选择本身就把判断传达给了会看数据的人，也不会牵着不看数据的人鼻子走。

**保留这些**（它们是图表的必要标注，不是旁白）：

- 模块标题：用**中性短语描述这张图是什么**（「播放量 × 三连率分布」「上榜次数分布」），不带问句、不带结论。
- 坐标轴标签、图例、数值、单位。
- 四象限 / 分组的**坐标定义**（「高播放 · 低三连」这种是在解释象限位置，属于图例性质，保留；但「吃了流量红利」这种价值判断要删）。
- 数据口径脚注（某指标怎么算的、统计范围）。

判断标准：这行字是在**解释图怎么读**（留），还是在**替读者下结论**（删）。

### 第六步：标清数据边界（诚实优先）

把这份数据**答不了的问题**在footer或角落用小字说清楚。比如缺某个分类字段就给不出按该分类的对比，缺转化/成本数据就给不出投入产出。

关键纪律：**数据没有的维度，不要用不严谨的方法硬造**（比如靠关键词猜分类、靠标题臆测属性）。宁可如实声明「要这个结论得补这类数据」，也不要给一个看着像、实则站不住的图。诚实的边界声明是专业度的体现，不是缺陷。

## 工程底线

- **数据与展示分离**：数据单独成文件，展示层引用它。换数据不动界面，界面迭代不碰数据。
- **可离线优先**：看板要能被随时打开。优先做成不依赖联网、不依赖构建步骤就能直接打开的形态；引第三方库前先确认它不会让看板在没网或换环境时打不开。
- **交互齐备**：维度切换、筛选、悬停看细节都要有；空状态、加载态、错误态都要有方案，别只画 happy path。
- **无障碍**：正文对比度达标；不靠颜色单一维度区分数据系列（配标签或形状）；尊重 reduced-motion；图例和悬停提示齐全。
- **先假后真（可选，降返工）**：布局没把握时，先用假数据把骨架搭出来确认版式和信息层级，再接真数据。能在画错之前发现布局问题。

## 边界（这个 skill 不做什么）

- **不做一次性分析报告**：要的是带 Insight 和行动建议的结论文档，用 business-analyst，不是本 skill。看板和报告是两种东西：报告替读者下结论，看板让读者自己读出结论。
- **不做产品功能拆解 / AC**：那是 product-breakdown。
- **不负责取数和数据清洗管道**：本 skill 从「已经有一份数据」开始。数据从哪来、怎么抽取，不在范围内（但加载时遇到的脏值要兜住，见第一步）。
- **不堆图**：模块数量跟着决策数量走。没有对应决策的图，再好看也不加。

## 语言规范（no_ai_style，对图表文案同样生效）

看板里所有给人读的文字（标题、标签、脚注），以及你跟用户的所有交流，都守这套规则。看板文案尤其要克制，它的天职是描述图表，不是表演。

- **不自我标榜**：不写「老实说」「说实话」「坦白讲」「不夸张地说」。
- **不解释自己用意**：不写「我这么做是为了……」「我的目的是……」。
- **不在结尾加引导式反问**：不写「要不要我……」「想不想我帮你……」。回答自然结束，缺信息就直说缺什么。
- **不附和**：不写「说得对」「确实」「没错」，直接接内容。
- **禁用破折号（——）和双引号（""）**：中文引用用「」。
- **不油腻煽情**：不写「掏心窝」「走心」「真心话」这类自我感动修饰。
- **不表态铺垫 / 自我宣誓**：不写「与其瞎说不如我去查」「让我直接……」这类宣告下一步的话，要做就直接做。
- **不贴写作手法标签**：不写「XX感做足」「XX式结构」「沉浸式」这类给自己手法贴标签的表述。
- **中文与数字/英文/代码关键词之间不加空格**：写「同时append」不写「同时 append」。
- **中文语境用中文标点**：用中文逗号、句号、冒号，不混英文逗号句号冒号（代码块除外）。
- **口语场景用口语**：给人读的判断和交流别用「闭环」「抓手」「颗粒度」「链路」这类书面腔精炼词，也别堆短促动词冒充生动。

核心：卸掉「想显得专业 / 想显得用心」的那股劲，把字写成人正常说话的样子。看板文案能少一个字就少一个字。
