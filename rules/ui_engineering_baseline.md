# 前端工程正确性底线清单

WCAG / Apple HIG / Material Design 的公认标准。所有界面方案和前端实现的硬门槛，不参与审美取舍排序。

使用方（按需Read，不常驻prompt）：

- ui-designer：步骤五方案自检时逐条对照，确认每条在方案里有交代
- code-reviewer：审查前端UI改动时作为工程底线检查依据

## 清单

- **可访问性**：正文对比度≥4.5:1（大字≥3:1）；focus ring可见；颜色不作唯一信息载体（配图标或文字）
- **触控**：可点击元素≥44×44px（移动端），相邻可点目标间距≥8px
- **表单**：label可见不靠placeholder代替；错误提示在字段下方就近显示；失焦校验不逐字符报错；必填有标识
- **导航**：返回行为可预测；底部tab≤5项；当前位置高亮
- **响应式**：移动端不横向滚动；正文移动端≥16px；断点系统一致（如375/768/1024/1440）
- **动效**：状态切换150到300ms；尊重`prefers-reduced-motion`；入场略慢出场略快
- **图表（如有）**：图例和tooltip齐全；不靠颜色单维区分数据系列；空/错/loading态都有方案
- **反馈与状态**：可交互元素有hover/active反馈；异步操作有loading态；弹窗有出入场动画；每个组件方案明确默认/hover/active/disabled四种状态
